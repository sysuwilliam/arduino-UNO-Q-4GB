# 知识点总结

## 1. Python 与 Arduino 的职责分离

这个例程最核心的知识点，是把系统分成了两个层面：

- Python 负责高层逻辑；
- Arduino 负责底层硬件执行。

这种分工方式在复杂项目里很常见，因为它能让代码更清晰，也更容易维护。

## 2. Router Bridge 的基本思路

Router Bridge 的本质可以理解为：

- Arduino 先注册一个可以被调用的函数；
- Python 再通过函数名发起远程调用；
- 桥接层负责完成中间的数据传输与函数分发。

在本例中：

- Arduino 端使用 `Bridge.provide("set_led_state", set_led_state)` 注册函数；
- Python 端使用 `Bridge.call("set_led_state", led_state)` 发起调用。

这是一种典型的“远程过程调用”思路。

## 3. GPIO 输出控制

Arduino 控制 LED 的根本方式是操作 GPIO 输出电平：

- `pinMode(LED_BUILTIN, OUTPUT)` 把引脚设为输出；
- `digitalWrite(...)` 给引脚写入高低电平。

这说明“LED 闪烁”的底层本质其实不是一个特殊功能，而是“定时修改一个输出引脚的电平”。

## 4. 板载 LED 反相逻辑

这个例程里一个很容易让初学者困惑的点是：

- `LOW` 表示亮；
- `HIGH` 表示灭。

这与许多人熟悉的“高电平点亮 LED”不同。

原因一般来自硬件连接方式，而不是代码错误。学习嵌入式时必须建立一个概念：

**最终行为要以电路连接方式为准，而不能只凭直觉判断高低电平含义。**

## 5. 状态切换的布尔逻辑

Python 端使用：

```python
led_state = not led_state
```

这是一个非常经典的布尔翻转写法。

适用场景包括：

- LED 闪烁；
- 开关状态切换；
- 模式切换；
- 标志位翻转。

这是本例最简洁也最重要的状态管理技巧之一。

## 6. 时间控制与阻塞式延时

本例通过：

```python
time.sleep(1)
```

实现每隔 1 秒切换一次状态。

这属于阻塞式延时，特点是：

- 写法简单；
- 逻辑直观；
- 适合教学和简单流程。

缺点是：

- 在等待期间当前逻辑不能做别的事情；
- 不适合复杂实时任务或多任务协作。

## 7. 最小可运行桥接例程的组成

从工程角度看，一个最小可运行的 Python-Arduino 桥接例程通常包括：

- 应用说明文档；
- App 元数据文件；
- Python 逻辑代码；
- Arduino 草图代码；
- 构建配置与依赖文件。

这个示例刚好完整地展示了这几部分，因此非常适合作为模板进行二次开发。

## 8. 依赖配置的重要性

`sketch.yaml` 中声明的 Router Bridge 及相关依赖说明了一件事：

**很多功能并不直接体现在业务源码里，但它们依然依赖构建系统和底层库的正确配置。**

初学者常常只盯着 `main.py` 和 `sketch.ino`，但实际上：

- 没有正确的平台配置，编译可能失败；
- 没有依赖库，桥接机制无法建立；
- 没有元数据文件，应用可能无法被平台正常识别。

## 9. 这个例程适合作为哪些能力的起点

学习完这个项目后，可以继续尝试：

- LED 闪烁频率可配置；
- Python 控制多个硬件输出；
- 传感器数据从 Arduino 回传到 Python；
- 桥接函数传递多个参数；
- 用图形界面或网络请求来触发 Arduino 动作。

## 10. 一句话总结

这个例程的本质是：

**Python 负责决定“什么时候切换”，Arduino 负责执行“怎么点亮或熄灭”，两者通过 Router Bridge 协同完成 LED 闪烁。**
