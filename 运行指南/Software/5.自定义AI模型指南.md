# Arduino App Lab 自定义 AI 模型

本教程教您如何创建和训练自己的 AI 模型来自定义 App Bricks。

**作者**：Christopher Méndez  
**最后修订**：2026年3月3日

---

## 概述

预定义模型为理解边缘 AI 提供了一个强大的起点。在本教程中，我们将通过设计和部署自己的自定义机器学习模型来扩展这些功能。通过转向自定义工作流程，我们可以让 Arduino App Lab 应用程序处理专门的任务和独特的数据集，从而根据我们的特定项目目标定制系统。

![Custom AI models](https://docs.arduino.cc/static/0f97c0cf164a5d1e4b910cc6da44cf70/a6d36/thumbnail-new.png)

通过训练自己的模型，您可以精确控制分类参数和性能指标，确保系统满足部署环境的具体要求，而不是依赖通用解决方案。

---

## 目标

- **收集**音频或图像相关模型的自定义数据集。
- **训练**在 Edge Impulse Studio 中从头开始的模型。
- **集成**您的自定义模型到 Arduino App Lab 以自定义您的 Bricks。

## 机器学习

为了设定背景，我们需要了解"AI 模型"实际上是什么。

在传统编程的世界里，我们编写明确的规则：*如果按下按钮 A，则打开 LED B。* 在**机器学习**中，我们不编写规则；我们提供示例。我们向计算机展示 100 张"香蕉"的照片和 100 张"苹果"的照片，计算机会自己找出区分它们的规则。该学习过程的结果就是一个**AI 模型**。

![Machine Learning](https://docs.arduino.cc/static/7e63eb1f8f3714bd2dbe48801b8484b0/a6d36/ml.png)

通过创建自定义模型，您本质上是在创建一个新的"大脑"文件，可以将其交换到 Arduino App Lab Bricks 中以完全改变它们的行为。

### Edge Impulse Studio

要创建这些自定义模型，我们使用 **Edge Impulse Studio**。

![Edge Impulse Studio](https://docs.arduino.cc/static/85aeea79882b8696585f8043d53c7acd/a6d36/edge-impulse.png)

Edge Impulse 是嵌入式机器学习的领先开发平台。把它想象成一个"实验室"，我们在那里准备 AI。它处理构建可以在 UNO Q 上运行的模型所需的整个流程。

#### 工作流程

您不需要编写代码来定义神经网络，而是使用 Studio 的可视化界面来引导流程：

![Edge Impulse workflow](https://docs.arduino.cc/static/062d27947366bda71b3d46d1a7c98f85/a6d36/workflow-2.png)

1. **数据采集：** 这是最关键的步骤。您收集/导入图像或音频样本到 Edge Impulse。您可以使用手机、计算机，甚至直接从 UNO Q 捕获数据。

2. **标注：** 数据上传后，您必须为每个样本分配一个类别或"标签"。此步骤为系统建立基本事实，准确教它哪些模式对应哪个输出类别。准确和一致的标注至关重要，因为它直接决定了模型学习识别什么。

3. **Impulse 设计：** 这是您构建"大脑"的地方。您定义一个**输入块**（例如音频或图像数据）、一个**处理块**（用于清理数据）和一个**学习块**（学习模式的神经网络）。

4. **训练：** Studio 使用其云服务器来处理数据。它将查看您的数据数千次，直到学会识别您定义的关键词或对象。

5. **部署：** Edge Impulse 允许我们专门为 **Arduino UNO Q** 导出训练好的模型，并直接导入到我们的 Arduino App Lab 应用程序中。

当您为 UNO Q 导出时，您会得到一个 **.eim（Edge Impulse Model）** 文件。此文件充当容器，保存运行模型所需的所有逻辑和训练参数。

## 创建自定义 AI 模型

- 在 Arduino App Lab 中，导航到 **My Apps** 并点击 **Create New App**。给它命名并保存。

- 在您的 App Python 文件中，通过复制粘贴以下内容更新 `main.py` 脚本：

```
from arduino.app_utils import App
from arduino.app_bricks.video_objectdetection import VideoObjectDetection

detection_stream = VideoObjectDetection(confidence=0.5, debounce_sec=0.0)

# Register a callback for when all objects are detected
def print_detections(detections: dict):
    for key, value in detections.items():
        for detection in value:
            entry = {
                "content": key,
                "confidence": f"{round(detection.get('confidence', 0) * 100)} %"
            }
            print(entry)

detection_stream.on_detect_all(print_detections)

App.run()
```

- 导航到左侧 Arduino App Lab 菜单中的 Bricks 部分，点击 **Add Brick** 按钮。选择 **Video Object Detection Brick**。

![Video Object Detection Brick](https://docs.arduino.cc/static/7c06ab38b27c0c93883ff95e85d2c722/a6d36/app-lab-0.5.png)

- 导航到 Brick 中的 **AI models** 选项卡。界面列出 Brick 可用的模型，如果没有训练新模型，则只显示内置的默认模型。

![AI models Brick option](https://docs.arduino.cc/static/04e13d6c2f9de6f4cb2dfc30b6c5c6c7/a6d36/app-lab-1.png)

- 要开始训练自定义模型，点击上图所示的 **Train new AI model**。然后点击 **Ok, let's start**。

![Start the Edge Impulse pairing](https://docs.arduino.cc/static/74a3b0fd2c486b3689a0c5e071259601/a6d36/app-lab-2.png)

- 如果这是您第一次，您将被引导完成 Arduino 账户创建或登录过程。

![Login Process](https://docs.arduino.cc/static/c8cf3e074ff8944058a47528e5de62bf/a6d36/app-lab-3.1.png)

- 您的同一 Arduino 账户将可用于登录 Edge Impulse Studio。

![Connect to Edge Impulse](https://docs.arduino.cc/static/4ea4d6abb975d75a822e46d65d7439fb/a6d36/app-lab-5.png)

- 系统将请求您授予 Arduino App Lab 访问 Edge Impulse 账户的权限。

![Edge Impulse permission](https://docs.arduino.cc/static/b39076d5937e118b4e6267dd771ead2b/a6d36/app-lab-5.5.png)

- Arduino 账户和 Edge Impulse 现在已连接，点击 **Start to Train your AI Model** 按钮。

![Start to train your AI model](https://docs.arduino.cc/static/ad376483efb4c682cf08cc1292ed73b2/a6d36/app-lab-6.png)

- 现在，您应该被重定向到 Edge Impulse Studio。

![Edge Impulse Studio](https://docs.arduino.cc/static/01e61db9d6d43a740c20f4bee4b75710/a6d36/app-lab-7.1.png)

### 基于图像的模型

要创建用于检测对象或分类图像的机器视觉模型，请按照以下步骤操作：

#### 新项目：

通过导航到您的个人资料图片（右上角）并点击 **Create new project** 创建您的第一个项目。选择一个与您项目目标相关的名称。

![New project creation](https://docs.arduino.cc/static/215defe753769ed4a22923e8c1be7ce7/a6d36/app-lab-8.png)

#### 类别：

定义您的模型应该检测的类别（例如 apple、banana）。我们将在创建数据集之前的标注过程中使用它们。

#### 数据集：

要训练模型，您首先需要数据。首先创建您想要检测的对象的数据集。

从项目 **Dashboard**，点击 **Collect new data**。您可以使用手机、计算机或 UNO Q 本身构建数据集，或直接上传现有图像。

![Collecting new data](https://docs.arduino.cc/static/9b21a91993cd899215927c2a59a01fdd/a6d36/app-lab-9.png)

为方便起见，我们将使用智能手机。扫描二维码打开链接，选择 **Camera** 选项，并授予必要的权限。

![QR code to connect your phone](https://docs.arduino.cc/static/bcc4171c10e062aacab54f49e8ef4e35/a6d36/app-lab-10.png)

为您想要检测的类别捕获各种图像（例如苹果和香蕉）。此外，Edge Impulse 将自动创建一个名为 **background** 的类别，根据您的图片教模型忽略什么。

![Apples and Bananas dataset](https://docs.arduino.cc/static/579a89e9ab893f78f42dab48ecf7a436/a6d36/app-lab-11.png)

***注意：您可以在捕获图像时标注它们，或者稍后使用 Edge Impulse 标注工具标注。***

#### Impulse 设计

在 **Create impulse** 部分创建您的 Impulse。在这里您将定义模型设置：

![Impulse design](https://docs.arduino.cc/static/a2017703b193c8a4e833cab00239cd4e/a6d36/app-lab-12.png)

- **图像分辨率：** 本例中为 320x320 像素
- **处理块：** Image
- **学习块：** Object Detection (Images)

点击 **Save Impulse** 并导航到 *Image processing block*。将 **Color depth** 参数保留为 `RGB`，然后点击 *Save parameters*，最后点击 *Generate features*。

#### 神经网络调优

为神经网络获取正确的设置需要时间和反复试验。按照以下步骤操作此模型：

- 导航到左侧菜单中的 Object detection 块。

- 通过点击"Choose a different model"将模型更改为 **MobileNetV2 SSD FPN-Lite 320x320**。

- 使用默认设置点击 **Save & Train**，等待训练性能结果。

![Training results](https://docs.arduino.cc/static/c9ce0533438b340f76a12538cd7d1e33/a6d36/app-lab-15.png)

**优化训练周期：**

默认设置为 **25 个周期**。监控训练输出。

- 如果准确率在第 15 或 20 个周期达到平台期，或者验证损失停止显著下降，您可以**减少**周期以节省未来运行的时间。

- 如果准确率仍在攀升，或者损失在第 25 个周期时仍在下降，**增加**周期数（例如增加到 40 或 50）以让模型完成学习。

**优化学习率：**

此模型使用 **0.15** 的较高默认学习率。

- 如果损失图波动剧烈（上下跳跃）或模型无法收敛，模型可能在"越过"最优权重。**降低**学习率（例如尝试 `0.1` 或 `0.05`）。

- 如果训练稳定但准确率仍然很低，您可以尝试稍微**增加**它，但要小心，因为此模型对高学习率敏感。

**防止过拟合：**

默认情况下，**数据增强**是**禁用**的。

- 如果您的模型在训练数据上表现完美（高准确率），但在将摄像头对准真实对象时失败（实际性能低），则模型"过拟合"了。

- 要解决此问题，**启用**数据增强。它在训练期间随机变换图像，强制模型学习一般特征而不是记住确切的像素值。

**检查设备约束：**

对象检测模型（如 SSD）计算量大。

- **推理时间：** 验证推理时间是否足够低以满足您的应用（例如 <500ms 可实现约 2 FPS）。

- **硬件限制：** 确保您的设备有足够的 RAM 来容纳模型。如果看到模型对 MCU 来说太大的警告，请验证您的特定硬件（如 Arduino UNO Q）具有运行它所需的内存。

在我们的案例中，默认设置给了我们良好的结果并已收敛：

- **推理时间：** 370 ms
- **Flash 使用：** 11 MB

![Training graphs](https://docs.arduino.cc/static/c17695d6727bde7ba235a1b3cd362ef9/a6d36/app-lab-16.png)

您可以从[这里](https://studio.edgeimpulse.com/public/846966/live)克隆本教程中使用的模型。

***这是一个数据集非常小的示例模型，用于演示目的。您可以通过修改它来改进它。***

**Impulse 设计效果：**

在 Edge Impulse 项目中，您会找到另一个名为 `Impulse #2` 的 impulse 设计。我们设置这个来说明一个观点。

![New performance results](https://docs.arduino.cc/static/d56a7aa965d3026b9e915921986c027f/a6d36/app-lab-25.png)

此 Impulse 具有以下设置：

- **分辨率：** 96x96 像素
- **神经网络架构：** FOMO (Faster Objects, More Objects) MobileNetV2 0.35

注意训练结果显示为混淆矩阵，使用这些设置我们实现了非常不同的性能：

- **推理时间：** 3 ms
- **峰值 RAM 使用：** 887.1 kB
- **Flash 使用：** 102.1 kB

***此模型的一个关键优势是它使用较低分辨率的输入图像，这显著减少了推理所需的计算资源。此外，与提供边界框的传统对象检测不同，FOMO 优化用于定位检测对象的中心点（质心）。***

#### 模型测试

要使用新数据测试模型的性能，请使用 **Live classification** 和 **Model testing** 部分。这些工具允许您验证模型在训练过程中未使用的图像中检测苹果和香蕉的能力。

![Live classification](https://docs.arduino.cc/static/8dae0860a3a168d87298f1b8849efb5b/a6d36/app-lab-17.png)

您还可以使用智能手机测试模型，使用我们创建数据集时使用的同一二维码（也可在 **Deployment** 部分找到）。这次，点击 **Switch to classification mode** 并等待模型下载和启动。最后，使用摄像头搜索一些苹果和香蕉。

![Model running on phone](https://docs.arduino.cc/static/ea52134ce288d3751ffbc136a2b15e73/a6d36/app-lab-18.png)

***模型已测试和验证后，返回 Arduino App Lab。***

## 自定义 AI 模型使用

返回 Arduino App Lab 后，您的新模型将出现在 Brick 的可用模型列表中。

![Custom model selection](https://docs.arduino.cc/static/ce3b7941a01025b2f9b05bdc922c69a2/a6d36/app-lab-21.1.png)

要在 App 中使用它，点击 **Install** 按钮，等待它在 Arduino UNO Q 上构建和安装。

![Model installation](https://docs.arduino.cc/static/9e7b26e34397cc0666573c76f9facb21/a6d36/app-lab-22.1.png)

最后，您可以通过点击 **Brick Configuration** 按钮选择新模型。

![Model selection](https://docs.arduino.cc/static/f6c5423a70996562da168ab20a7da92f/a6d36/app-lab-23.1.png)

### 运行 App

现在运行您的 App，它将使用您的自定义 AI 模型来检测苹果和香蕉。

![Run the App](https://docs.arduino.cc/static/5f44de8dcf08d7e905fb1211c4e70001/a6d36/final-image.png)

***在 Python 控制台中，您将看到检测日志及其置信度。***

此外，您可以通过在您喜欢的浏览器中导航到 `<UNO-Q-IP-ADDRESS>:4912` 来预览摄像头实时画面并实时查看模型运行。

## 结论

在本教程中，您学习了如何通过使用 Edge Impulse Studio 设计和部署自定义 AI 模型来扩展 Arduino App Lab 的功能。您探索了完整的机器学习流程——从收集自定义图像数据集到训练为 Arduino UNO Q 优化的 MobileNetV2 SSD 对象检测模型。

由于 Arduino App Lab 和 Edge Impulse 之间的无缝集成，您现在可以将通用"模型"替换为专门的模型，使您的 Bricks 能够高精度识别特定对象（如苹果和香蕉）。这将您的 UNO Q 从简单的计算机转变为能够解决独特现实问题的定制边缘 AI 设备。

- 从 Edge Impulse 手动导出 `.eim` 文件，以在 Arduino App Lab 之外的 C++ 或 Python 项目中尝试运行自定义模型。
