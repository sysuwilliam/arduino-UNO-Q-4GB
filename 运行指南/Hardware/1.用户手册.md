## 产品概述

Arduino UNO Q 为 Arduino 生态系统解锁了新的性能水平，融合了 Qualcomm 先进的 QRB2210 微处理器（MPU）的强大计算能力——运行完整 Debian Linux 操作系统并具有上游支持——以及专用 STM32U585 微控制器（MCU）的实时响应能力——在 Zephyr OS 上运行 Arduino sketches——全部集成在一块单板计算机上。

### 板卡架构概述

Arduino UNO Q 将运行完整 Linux 环境的高性能 Qualcomm® QRB2210 MPU 与运行在单片紧凑板卡上的 STMicroelectronics® STM32U585（32位 Arm® Cortex®-M33）MCU 的实时精度相结合。这种混合架构为 AIoT、机器学习和高级自动化应用提供了所需的强大性能和响应能力。

![UNO Q 主要组件](https://docs.arduino.cc/static/567189fab6cc1a00404d38e37b42e755/a6d36/uno-q-architecture-3.png)

如上图所示，以下是板卡主要组件的概述：

- **微处理器（MPU）**：Qualcomm® QRB2210 是一款四核 Arm® Cortex®-A53 处理器，运行频率为 2.0 GHz，配备 Adreno™ 702 GPU（845 MHz）用于 3D 图形加速，以及双 ISP，支持高达 25 MP @ 30 fps。它运行具有上游支持的 Debian Linux OS，非常适合嵌入式视觉和边缘计算应用。

- **微控制器（MCU）**：STM32U585 微控制器具有 Arm® Cortex®-M33 核心，运行频率高达 160 MHz，配备 2 MB Flash 存储器和 786 KB SRAM。它运行 Zephyr OS，为低功耗嵌入式应用提供安全高效的平台。

- **无线连接**：WCBN3536A 无线模块提供双频 Wi-Fi® 5（2.4/5 GHz）和 Bluetooth® 5.1 连接，均配备板载天线以实现可靠的无线性能。

- **存储器**：板卡提供 16 GB 或 32 GB eMMC 存储选项，以及 2 GB 或 4 GB LPDDR4 RAM 选项，为嵌入式应用提供快速的存储器访问和可靠的存储。

- **多媒体编解码器**：ANX7625 多媒体编解码器通过板载 USB-C 连接器实现视频和音频输出，为嵌入式应用中的显示和声音传输提供高速接口。

- **电源管理**：UNO Q 包含 Qualcomm® PM4145 电源管理集成电路（PMIC），以满足始终连接的物联网设备的需求。

## 首次使用

### 为板卡供电

Arduino UNO Q 可通过以下方式供电：

- 提供 5 VDC 3 A 的 USB-C® 线缆（不含）
- 连接到 5V 引脚的外部 +5 VDC 电源
- 连接到 VIN 引脚的外部 +7-24 VDC 电源

![UNO Q 供电选项](https://docs.arduino.cc/static/72456f6873252eb705cfd28538166e8a/a6d36/power-options-3.png)

> 点击[这里](https://docs.arduino.cc/tutorials/uno-q/power-specification/)了解更多关于 UNO Q 电源规格的信息。

### 安装 Arduino App Lab

[Arduino App Lab](https://docs.arduino.cc/software/app-lab/) 是一个统一的开发环境，将经典 Arduino 体验扩展到高性能计算领域。Arduino App Lab 让您可以无缝地将 Arduino sketches、Python 脚本和容器化的 Linux 应用程序组合到单一工作流程中。

通过称为 Bricks 的代码构建块、预配置的 AI 模型和集成编排，它降低了复杂性，同时使您能够创建从简单原型到高级、计算密集型应用的所有内容。

![Arduino App Lab](https://docs.arduino.cc/static/782ed8393ae066932b79a19418651a50/a6d36/app-lab.png)

Arduino App Lab **预装**在 UNO Q 上，可以在单板计算机（SBC）模式下使用。我们强烈建议使用 **4 GB RAM** 的 UNO Q 版本以获得更好的**独立**体验。

![SBC 和 PC 托管模式](https://docs.arduino.cc/static/a149a5e406178f25376d784b1d615e6d/a6d36/modes-2.png)

要在个人电脑上安装以进行 **PC 托管**设置，请访问我们官方网站的[软件部分](https://www.arduino.cc/en/software/#app-lab-section)，滚动到 Arduino App Lab 并选择您操作系统对应的版本。

![Arduino App Lab 安装](https://docs.arduino.cc/static/34c61616693c68c4b2df9b2bf7aa3a78/a6d36/app-lab-download.png)

#### 网络模式

即使您已将 Arduino UNO Q 设置为**单板计算机**，您也可以使用 Arduino App Lab 桌面版和**网络模式**从个人机器远程访问它。两种模式可以同时使用。

- 当您打开 Arduino App Lab 时，如果您的板卡可以在本地网络上被发现，它将显示 **Network** 标签。
- 点击它并输入 Linux 密码登录。
- 您现在可以远程访问板卡。

![网络模式演示](https://docs.arduino.cc/4bd91b938d5f092636a90406db84e42d/network-mode.gif)

通过此方法，您可以从本地网络中的任何机器访问您的 UNO Q。这允许您像直接连接到板卡一样使用 Arduino App Lab，您可以像通过 USB-C® 连接一样开发和运行 Apps。

网络模式依赖**本地网络发现（mDNS）**自动查找同一网络上的板卡。某些网络配置（如访客 Wi-Fi、企业或物联网网络、VPN 或严格的防火墙规则）可能会阻止自动发现，即使板卡已连接到 Wi-Fi。

**发现问题故障排除**

- **Windows 用户：** 首次启动 Arduino App Lab 时，您可能会收到 Windows Defender（或其他安全软件）关于 `mdns-discovery.exe` 的提示。您必须**允许**此访问才能发现板卡。*注意：在已经运行过 Arduino IDE 的系统上，提示可能不会出现。*

- **防火墙设置：** 如果板卡未显示，请确保您的防火墙允许 **UDP 端口 5353** 上的流量，这是 mDNS 发现所需的。

> **注意：** 能够通过浏览器、SSH 或 IP 地址访问板卡并不保证它会在网络模式中出现。Arduino App Lab 使用本地网络发现自动列出板卡。

### Linux 主机设置（Linux 用户必需）

如果您从 Linux 主机使用 UNO Q，您必须为您的用户账户配置 USB 设备权限。如果没有适当的权限，多个操作可能会失败，导致令人沮丧的错误。

当 Arduino App Lab 尝试通过 USB 连接到板卡时，它会静默失败。您可能需要检查 DevTools 控制台才能看到关于设备权限不足的实际错误消息。

同样，如果您尝试刷写新镜像到板卡，过程将失败并显示以下错误：

![udev 规则错误 (1)](https://docs.arduino.cc/static/6f7a1796ae11d9a5993ff1d6f2521d90/a6d36/udev_rules_1.png)

尝试通过 ADB 命令（如 `adb shell`）与板卡通信时也会出现相同的错误。

#### 理解权限

默认情况下，Linux 上的 USB 设备仅可由 root 用户或特定组中的用户访问。当 UNO Q 连接到您的计算机时，它会创建 USB 设备文件，通常位于以下目录：

```
/dev/bus/usb/
```

该目录需要写入权限才能进行通信。

UNO Q 支持两种 USB 模式，每种都有自己的 USB 标识符。在正常运行模式下，板卡使用 USB VID `2341` 和 PID `0078`。

当您需要刷写板卡固件时，它会进入**紧急下载模式（EDL）**，使用 USB VID `05c6` 和 PID `9008`。两种模式都需要正确的权限才能正常工作。

#### 安装 udev 规则

为您的用户账户配置必要权限的正确方法是通过 **udev 规则**。`udev` 是 Linux 子系统，管理 `/dev` 目录中的设备节点。以下 udev 规则将授予您的用户账户访问 UNO Q 的权限：

```
# 运行模式
SUBSYSTEMS=="usb", ATTRS{idVendor}=="2341", ATTRS{idProduct}=="0078", MODE="0660", TAG+="uaccess"
# EDL 模式
SUBSYSTEMS=="usb", ATTRS{idVendor}=="05c6", ATTRS{idProduct}=="9008", MODE="0660", TAG+="uaccess"
```

运行以下命令安装必要的 udev 规则：

```bash
echo \
'# Operating mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="2341", ATTRS{idProduct}=="0078", MODE="0660", TAG+="uaccess"
# EDL mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="05c6", ATTRS{idProduct}=="9008", MODE="0660", TAG+="uaccess"' \
| \
  sudo \
 tee \
      "/etc/udev/rules.d/60-Arduino-UNO-Q.rules" \
&& \
sudo \
 udevadm control \
    --reload-rules \
&& \
sudo \
 udevadm trigger
```

![udev 规则 (2)](https://docs.arduino.cc/static/31b990a2b0b3ec1d1bf67e0dfae97d44/a6d36/udev_rules_2.png)

此命令按顺序执行多个操作。首先，它创建规则文件并将 udev 规则写入：

```
/etc/udev/rules.d/60-Arduino-UNO-Q.rules
```

`60-` 前缀决定规则处理顺序，数字越小表示优先级越高。`/etc/udev/rules.d/` 目录中的文件是用户特定的覆盖，在系统更新后保留。

每条规则根据特定条件匹配 USB 设备。`SUBSYSTEMS=="usb"` 参数使规则仅适用于 USB 设备。

`ATTRS{idVendor}` 和 `ATTRS{idProduct}` 参数通过其唯一的 USB 供应商和产品 ID 识别 UNO Q。`MODE="0660"` 参数设置设备文件权限，允许所有者和组读写设备。

`TAG+="uaccess"` 参数通过 `systemd-logind` 授予当前登录用户访问权限。

创建规则文件后，命令通过运行以下命令重新加载所有 udev 规则：

```bash
udevadm control --reload-rules
```

![udev 规则 (3)](https://docs.arduino.cc/static/54d68b010d7421e324e528d590639024/a6d36/udev_rules_3.png)

这告诉 udev 重新加载所有规则文件，无需重启系统。

`udevadm trigger` 使用新加载的规则重新处理所有现有设备，确保任何当前连接的 UNO Q 板卡获得正确的权限。

#### 验证安装

使用以下命令验证规则是否正确安装：

```bash
cat /etc/udev/rules.d/60-Arduino-UNO-Q.rules
```

![udev 规则 (4)](https://docs.arduino.cc/static/21c2b71b69854468cbfd782098a0642b/a6d36/udev_rules_4.png)

这应该显示您刚刚安装的两条 udev 规则。

要检查 UNO Q 是否被检测到，可以使用以下命令：

```bash
lsusb | grep -E "2341:0078|05c6:9008"
```

当板卡以运行模式连接时，您应该看到类似以下输出：

![udev 规则 (5)](https://docs.arduino.cc/static/c25ed64c4d33e7c00665e0180fbdc298/a6d36/udev_rules_5.png)

#### 应用更改

安装 udev 规则后，**您需要断开并重新连接板卡才能使更改生效**。

从计算机上拔下 UNO Q，等待几秒钟。然后重新连接板卡。当板卡重新连接时，新权限将自动应用。

如果您在安装规则时有任何应用程序打开（如 Arduino App Lab 或运行 ADB 命令的终端），您应该在重新连接板卡后关闭并重新启动它们。这允许应用程序以其新权限正确检测板卡。

对于 ADB 命令，可以使用以下命令：

```bash
adb devices
```

此命令列出连接到系统的所有 Android Debug Bridge 设备。您的 UNO Q 应出现在列表中，没有任何错误消息。

以下命令可用于访问 UNO Q：

```bash
adb shell
```

![udev 规则 (6)](https://docs.arduino.cc/static/2367909092d869f4d4ccf0817715652c/a6d36/udev_rules_6.png)

#### 替代安装方法

您可以使用 [*Arduino Core Zephyr 仓库*](https://github.com/arduino/ArduinoCore-zephyr)中的官方 `post-install` 脚本自动配置这些权限。此脚本执行与上述手动方法相同的 udev 规则安装。

要下载并运行 `post-install` 脚本，导航到您的下载目录并使用 `wget` 从仓库获取脚本：

```bash
cd ~/Downloads
wget https://raw.githubusercontent.com/arduino/ArduinoCore-zephyr/main/post_install.sh
```

![udev 规则 (7)](https://docs.arduino.cc/static/41d9f49d7b9fa525d5d58b2742a6d135/a6d36/udev_rules_7.png)

```bash
chmod +x post_install.sh
sudo ./post_install.sh
```

![udev 规则 (8)](https://docs.arduino.cc/static/36a3e8d280f595e27a5870887c9d42af/a6d36/udev_rules_8.png)

如果您的系统上没有 wget，可以使用 `curl` 代替：

```bash
cd ~/Downloads
curl -O https://raw.githubusercontent.com/arduino/ArduinoCore-zephyr/main/post_install.sh
chmod +x post_install.sh
sudo ./post_install.sh
```

或者，您可以克隆整个仓库并从那里运行脚本：

```bash
cd ~/Downloads
git clone https://github.com/arduino/ArduinoCore-zephyr.git
cd ArduinoCore-zephyr
chmod +x post_install.sh
sudo ./post_install.sh
```

当提示时，输入您的密码。脚本将创建 udev 规则文件、重新加载 udev 规则，并将其应用于任何当前连接的设备，就像手动安装方法一样。

运行脚本后，您可以使用手动安装部分中显示的相同命令验证安装：

```bash
cat /etc/udev/rules.d/60-arduino-zephyr.rules
```

![udev 规则 (9)](https://docs.arduino.cc/static/15ab7e2f9d6bcdb59abd315d694acf3e/a6d36/udev_rules_9.png)

请注意，脚本创建的文件名为 `60-arduino-zephyr.rules`，而不是 `60-Arduino-UNO-Q.rules`，但它具有相同权限规则，适用于两种运行模式。

使用任一方法安装 udev 规则后，**将您的 UNO Q 板卡从计算机上拔下，等待几秒钟，然后重新连接**。这允许应用新权限。重新连接后，您可以继续刷写过程的下一步。

### 启动时运行 App

您可以配置特定的 App 在 UNO Q 上电时自动启动。这对于板卡在没有计算机连接的情况下运行的独立项目很有用。

> **注意：** 您不能直接从 UI 将内置 **Example** 设置为启动应用。您必须先从示例中点击 **Copy and edit app**，或从头创建新 **App**。

1. 打开您的自定义 App（或示例的副本）。
2. 找到右上角的 **Run** 按钮。
3. 点击 Run 按钮旁边的箭头（▼）打开菜单。
4. 将 **Run at startup** 开关切换到 **ON** 位置。

![启动时运行选项](https://docs.arduino.cc/static/2eaa971d3cf57c2e275de703099db401/a6d36/run-at-startup.png)

配置后，您的 App 名称旁边会出现 **DEFAULT** 标记，表示它将在启动时自动运行。

#### 高级：使用 CLI

或者，您可以在 UNO Q 终端内使用命令行界面（CLI）设置默认应用：

### RGB LED

UNO Q 配备 4 个 RGB LED。其中两个由 Qualcomm 微处理器连接和控制，另外两个由 STM32 微控制器控制。

![UNO Q RGB LED](https://docs.arduino.cc/static/10826219b80fd6d653c4601862558c1e/a6d36/rgb-led.png)

#### MPU 控制的 LED

LED #1 和 #2 由 MPU 控制。

我们的 Linux OS 中有专用的 LED 接口来控制这些 LED，可以通过 **命令行**、使用 **SSH**、从 PC 终端的 **ADB** 连接，或在单板计算机模式下使用 Linux 内置终端应用程序来控制：

```bash
echo 1 | tee /sys/class/leds/red:user/brightness # 设置 HIGH/ON
echo 0 | tee /sys/class/leds/red:user/brightness # 设置 LOW/OFF
```

![Linux LED 控制](https://docs.arduino.cc/df8bc8546711790b5eeecc1e38706d59/linux-led-control.gif)

LED 颜色段定义如下：

**LED 1：**
- **红色：** `red:user`
- **绿色：** `green:user`
- **蓝色：** `blue:user`

**LED 2：**
- **红色：** `red:panic`
- **绿色：** `green:wlan`
- **蓝色：** `blue:bt`

> LED 2 用于显示系统状态：`PANIC`、`WLAN` 和 `BT`。但它也可以由用户控制。

您也可以从 Python 脚本控制这些 LED。记得在 Arduino App Lab 中**创建新 App**，然后将以下脚本复制并粘贴到您 App 的 python 部分：

```python
import time

from arduino.app_utils import App

LED1_R = "/sys/class/leds/red:user/brightness"
LED1_G = "/sys/class/leds/green:user/brightness"
LED1_B = "/sys/class/leds/blue:user/brightness"

LED2_R = "/sys/class/leds/red:panic/brightness"
LED2_G = "/sys/class/leds/green:wlan/brightness"
LED2_B = "/sys/class/leds/blue:bt/brightness"

def set_led_brightness(led_file, value):
    try:
        with open(led_file, "w") as f:
            f.write(f"{value}\n")
    except Exception as e:
        print(f"Error writing to {led_file}: {e}")

# 关闭所有 LED
set_led_brightness(LED1_R, 0)
set_led_brightness(LED1_G, 0)
set_led_brightness(LED1_B, 0)
set_led_brightness(LED2_R, 0)
set_led_brightness(LED2_G, 0)
set_led_brightness(LED2_B, 0)

def loop():
    # 闪烁 LED 1 红色段
    set_led_brightness(LED1_R, 1)
    time.sleep(1)
    set_led_brightness(LED1_R, 0)
    time.sleep(1)

App.run(user_loop=loop)
```

您也可以使用专用的 Linux 模块 `Leds` 控制这些 LED：

```python
# 参数分别对应 R、G 和 B 颜色
Leds.set_led1_color(1,0,0) # LED 1 显示红色
Leds.set_led2_color(1,0,0) # LED 2 显示红色
```

记得在 Arduino App Lab 中**创建新 App**，然后将以下脚本复制并粘贴到您 App 的 python 部分：

```python

    # 关闭 LED (0, 0, 0)
    Leds.set_led1_color(0,0,0)

```

void setup(){

  pinMode(LED3_G, OUTPUT);

void loop(){

  digitalWrite(LED3_G, HIGH);
  digitalWrite(LED3_B, HIGH);
  delay(1000);
  digitalWrite(LED3_R, HIGH);
  digitalWrite(LED3_G, LOW);   // 打开绿色段
  digitalWrite(LED3_B, HIGH);
  delay(1000);
  digitalWrite(LED3_R, HIGH);
  digitalWrite(LED3_G, HIGH);
  digitalWrite(LED3_B, LOW);   // 打开蓝色段
  delay(1000);
}

### 硬件调试 UART 接口

UNO Q 提供专用的低级 UART 接口用于调试和系统诊断。此接口直接连接到 SoC 的主控制台（TTY），允许您观察引导和内核日志、排查系统问题，或在 SSH 或 ADB 等网络服务可用之前访问 shell 环境。

![调试 shell](https://docs.arduino.cc/40e41d2593a4441e3ceb16d3be558d40/debug-shell.gif)

此接口通过 UNO Q 上的 JCTL 连接器提供。有关详细信息，请参阅[引脚图](#引脚图)部分，并按照上面的接线示例访问它。

> ⚠️ **警告：** 此接口以 1.8 V 逻辑电平工作，必须与兼容的 USB 转 TTL 转换器一起使用以避免硬件损坏。

#### 先决条件

- 1.8V USB 转 TTL 转换器（例如 DSD Tech SH-U09C5）
- 为 UNO Q 供电的 USB-C 线缆
- 串行终端（例如 [Tera Term](https://teratermproject.github.io/index-en.html)）

#### UART 参数：

- **波特率：** 115200 bps
- **逻辑电平：** 1.8 V

此控制台提供对引导加载程序（例如 SPL 和 U-Boot）打印的低级系统消息的访问，这些消息通过 SSH 或其他高级接口不可见。例如，它允许捕获与电源传输协商或早期引导阶段的硬件初始化相关的日志，提供否则无法访问的信息。

它可用于登录板卡并通过 shell 与系统交互。使用板卡设置过程中配置的 **Linux 凭据**进行身份验证。

![Shell 登录](https://docs.arduino.cc/static/c16a31399c920ef409c4c1a73c40da40/a6d36/debug-shell-2.png)

## 引脚

UNO Q 配备两种类型的连接器：顶部的**经典 UNO 风格接头**，设计用于原型设计和调试，保证与 Arduino UNO 扩展板完全兼容；底部的**高速接头连接器**，专门设计用于与 UNO Q 载板集成。

### 数字引脚

UNO Q 有 47 个由 **STM32 微控制器**控制的数字引脚，其中 22 个通过 UNO 风格连接器暴露，25 个通过 JMISC 连接器暴露，映射如下：

| 微控制器引脚 | Arduino 引脚映射 | 引脚功能 |
|--------------|------------------|----------|
| PB7 | D0 / RX | GPIO / UART RX |
| PB6 | D1 / TX | GPIO / UART TX |
| PB3 | D2 | GPIO |
| PB0 | D3 | GPIO / OPAMP OUT |
| PA12 | D4 / FDCAN1_TX | GPIO / CAN 总线 TX |
| PA11 | D5 / FDCAN1_RX | GPIO / CAN 总线 RX |
| PB1 | D6 | GPIO |
| PB2 | D7 | GPIO |
| PB4 | D8 | GPIO |
| PB8 | D9 | GPIO |
| PB9 | D10 / SS | GPIO / SPI SS |
| PB15 | D11 / MOSI | GPIO / SPI MOSI |
| PB14 | D12 / MISO | GPIO / SPI MISO |
| PB13 | D13 / SCK | GPIO / SPI SCK |
| PA4 | D14 / DAC0 | GPIO / ADC / DAC |
| PA5 | D15 / DAC1 | GPIO / ADC / DAC |
| PA6 | D16 | GPIO / ADC / OPAMP IN + |
| PA7 | D17 | GPIO / ADC / OPAMP IN - |
| PC1 | D18 / SDA2 | GPIO / ADC / I2C SDA |
| PC0 | D19 / SCL2 | GPIO / ADC / I2C SCL |
| PB11 | D20 / SDA | GPIO / I2C SDA |
| PB10 | D21 / SCL | GPIO / I2C SCL |

> **注意：** 引脚 D14 到 D19 也具有模拟功能。

UNO Q 的数字引脚可以通过 Arduino 编程语言的内置函数用作输入或输出。

数字引脚的配置在 `setup()` 函数中使用内置函数 `pinMode()` 完成，如下所示：

```cpp
// 引脚配置为输入
pinMode(pin, INPUT);
// 引脚配置为输出
pinMode(pin, OUTPUT);
// 引脚配置为输入，启用内部上拉电阻
pinMode(pin, INPUT_PULLUP);
```

配置为输入的数字引脚的状态可以使用内置函数 `digitalRead()` 读取，如下所示：

```cpp
// 读取引脚状态，将值存储在状态变量中
state = digitalRead(pin);
```

配置为输出的数字引脚的状态可以使用内置函数 `digitalWrite()` 更改，如下所示：

```cpp
// 设置引脚为高电平
digitalWrite(pin, HIGH);
// 设置引脚为低电平
digitalWrite(pin, LOW);
```

下面显示的示例代码使用数字引脚 `D5` 控制 LED，并读取连接到数字引脚 `D4` 的按钮状态：

![数字 I/O 示例接线](https://docs.arduino.cc/static/f8b3b793fc7770c941b38c0c89a68401/a6d36/digital-io.png)

1. 在 Arduino App Lab 中创建新 App。

   ![创建新 app](https://docs.arduino.cc/static/3970781efa4ef9cd3457a952046f05b2/a6d36/create-app.png)

2. 通过点击 **Add Sketch Library** 并搜索来安装 **Arduino_RouterBridge** 库。

   ![库安装](https://docs.arduino.cc/static/a3f6680e10a424e991fdfaa926d8b09e/a6d36/lib-install-app-lab.png)

3. 将以下示例复制并粘贴到您新 App 的 "sketch" 部分。

```cpp
#include <Arduino_RouterBridge.h>
// 定义按钮和 LED 引脚
int buttonPin = D4;
int ledPin = D5;

// 存储按钮状态的变量
int buttonState = 0;

void setup() {
  // 配置按钮和 LED 引脚
  pinMode(buttonPin, INPUT_PULLUP);
  pinMode(ledPin, OUTPUT);

  // 初始化串口通信
  Monitor.begin();
}

void loop() {
  // 读取按钮状态
  buttonState = digitalRead(buttonPin);

  // 如果按下按钮，打开 LED 并将其状态打印到串口监视器
  if (buttonState == LOW) {
    digitalWrite(ledPin, HIGH);
    Monitor.println("- Button is pressed. LED is on.");
  } else {
    // 如果未按下按钮，关闭 LED 并打印到串口监视器
    digitalWrite(ledPin, LOW);
    Monitor.println("- Button is not pressed. LED is off.");
  }

  // 等待 1000 毫秒
  delay(1000);
}
```

### 模拟引脚

UNO Q 在 **JANALOG** 连接器中具有众所周知的模拟引脚；更多详情如下：

#### 模数转换器（ADC）

在 **JANALOG** 连接器中，UNO Q 有 6 个 14 位 ADC 引脚，映射如下：

| 微控制器引脚 | Arduino 引脚映射 | 引脚功能 |
|--------------|------------------|----------|
| PA4 | A0 | GPIO / ADC / DAC |
| PA5 | A1 | GPIO / ADC / DAC |
| PA6 | A2 | GPIO / ADC / OPAMP IN + |
| PA7 | A3 | GPIO / ADC / OPAMP IN - |
| PC1 | A4 | GPIO / ADC / I2C SDA |
| PC0 | A5 | GPIO / ADC / I2C SCL |

模拟输入引脚可以通过 Arduino 编程语言的内置函数使用。

UNO Q ADC **分辨率**可以使用 `analogReadResolution(bits)` 函数在 14、12、10 或 8 位之间配置：

```cpp
// ADC 分辨率设置为 14 位（0 到 16383）
analogReadResolution(14);
```

默认 ADC **电压参考**为 3.3V，可以使用 `analogReference()` 函数通过以下参数通过软件更改：

| 模拟电压参考（VREF+） | 参数 | 来源 |
|----------------------|------|------|
| 1.5 V | AR_INTERNAL1V5 | 内部 |
| 1.8 V | AR_INTERNAL1V8 | 内部 |
| 2.048 V | AR_INTERNAL2V05 | 内部 |
| 2.5 V | AR_INTERNAL2V5 | 内部 |
| 2 V ~ VDD | AR_EXTERNAL | 外部 |

> 当使用 `AR_EXTERNAL` 参考时，可以通过 AREF 引脚提供外部电压参考。

要设置与默认不同的模拟参考，请参阅以下示例：

```cpp
analogReference(AR_INTERNAL2V5);
```

![ADC 输入示例接线](https://docs.arduino.cc/static/f4d1d479256a906ce82b9638dbf528d3/a6d36/analog-adc.png)

   ![库安装](https://docs.arduino.cc/static/a3f6680e10a424e991fdfaa926d8b09e/a6d36/lib-install-app-lab.png)

3. 将以下示例复制并粘贴到您新 App 的 "sketch" 部分。

#### 数模转换器（DAC）

UNO Q 有两个 DAC 输出，映射如下：

| 微控制器引脚 | Arduino 引脚映射 | 引脚功能 |
|--------------|------------------|----------|
| PA4 | DAC0 | GPIO / ADC / DAC |
| PA5 | DAC1 | GPIO / ADC / DAC |

UNO Q 的数模转换器可以通过 Arduino 编程语言的内置函数输出模拟电压。

DAC 输出分辨率可以使用 `analogWriteResolution()` 函数从 8 位配置到 12 位，如下所示：

```cpp
// DAC 分辨率设置为 12 位（0 到 4095）
analogWriteResolution(12);  // 输入所需的分辨率位数（8, 10, 12）
```

要通过 DAC 引脚输出模拟电压值，请使用 `analogWrite()` 函数，以 DAC 通道作为参数。请参阅以下示例：

```cpp
analogWrite(DAC0, value);   // 值应在 DAC 分辨率范围内（例如 12 位分辨率为 0-4095）
```

> 如果将普通 GPIO 传递给 `analogWrite()` 函数，输出将是 PWM 信号。

以下 sketch 将在 UNO Q 的 `A0/DAC0` 引脚上创建 **60 Hz 正弦波**信号：

1. 在 Arduino App Lab 中创建新 App。

   ![创建新 app](https://docs.arduino.cc/static/3970781efa4ef9cd3457a952046f05b2/a6d36/create-app.png)

2. 将以下示例复制并粘贴到您新 App 的 "sketch" 部分。

```cpp
const float freq = 60.0f;
const int N = 256;     // 256 采样/周期
const uint32_t Ts_us = (uint32_t)llroundf(1e6f / (freq * N));

uint16_t lut[N]; // 在此处存储正弦波

void setup() {
  analogWriteResolution(12);

  for (int i = 0; i < N; ++i){
    lut[i] = 2048 + (1000.0 * sin(2 * PI * i / N));
  }
}

void loop() {
  static uint32_t t_next = micros();
  for (int i = 0; i < N; ++i) {
    analogWrite(DAC0, lut[i]);  // 输出正弦波值
    t_next += Ts_us;
    while ((int32_t)(micros() - t_next) < 0) { /* spin */ }
  }
}
```

### PWM 引脚

UNO Q 有 6 个 PWM（脉冲宽度调制）引脚，映射如下：

| 微控制器引脚 | Arduino 引脚映射 | 引脚功能 |
|--------------|------------------|----------|
| PB0 | D3 | GPIO / OPAMP OUT / PWM |
| PA11 | D5 / FDCAN1_RX | GPIO / CAN 总线 RX / PWM |
| PB1 | D6 | GPIO / PWM |
| PB8 | D9 | GPIO / PWM |
| PB9 | D10 / SS | GPIO / SPI SS / PWM |
| PB15 | D11 / MOSI | GPIO / SPI MOSI / PWM |

此功能可以使用内置函数 `analogWrite()` 使用，如下所示：

```cpp
analogWrite(pin, value);
```

默认情况下，输出分辨率为 **8 位**，因此输出值应在 0 到 255 之间。要设置更大的分辨率，请使用内置函数 `analogWriteResolution`，如下所示：

```cpp
// PWM 分辨率设置为 10 位（0 到 1023）
analogWriteResolution(10);
```

以下是如何创建可变占空比 PWM 信号的示例：

1. 在 Arduino App Lab 中创建新 App。

   ![创建新 app](https://docs.arduino.cc/static/3970781efa4ef9cd3457a952046f05b2/a6d36/create-app.png)

2. 将以下示例复制并粘贴到您新 App 的 "sketch" 部分。

```cpp
const int analogInPin = A0;  // 电位器连接的模拟输入引脚
const int pwmOutPin = D3;    // PWM 输出引脚

int sensorValue = 0;  // 从电位器读取的值
int outputValue = 0;  // 输出到 PWM 的值（模拟输出）

void setup() {
  // 定义 PWM 输出分辨率
  analogWriteResolution(10);  // 0 - 1023 -> 0 - 100% 占空比
  analogReadResolution(14);   // 0 - 16383
}

void loop() {
  // 读取模拟输入值：
  sensorValue = analogRead(analogInPin);
  // 将其映射到模拟输出范围：
  outputValue = map(sensorValue, 0, 16383, 0, 1024);
  // 更改模拟输出值：
  analogWrite(pwmOutPin, outputValue);

  // 在下一个循环前等待 2 毫秒
  // 以便 ADC 在最后一次读取后稳定：
  delay(2);
}
```

现在您可以通过旋转电位器来控制 PWM 信号占空比。

![PWM 输出信号](https://docs.arduino.cc/static/01c4c7cb4a70a9e121032c42e61b1c89/a6d36/pwm-output.png)

> PWM 频率固定为 500 Hz。

## 通信

本节介绍 Arduino UNO Q 支持的不同通信协议。

### Bridge - 远程过程调用（RPC）库

Arduino UNO Q 使用 RPC（远程过程调用）在 Linux（Qualcomm MPU）端和实时 STM32 MCU 之间交换数据。此机制允许在一个处理器上运行的函数被另一个处理器透明地调用，就像它们是本地调用一样。

![UNO Q RPC](https://docs.arduino.cc/static/316b6159bfceb0b529f503c47547041c/a6d36/rpc.png)

#### 概述

`Bridge` 库提供构建在 `Arduino_RPClite` 框架之上的通信层。它管理 MPU 和 MCU 之间的双向 RPC 流量，处理方法绑定、请求转发和异步响应。

- **MPU 端（Qualcomm QRB，Linux）**：运行更高级的服务，可以远程调用 MCU 函数。

- **MCU 端（STM32，Zephyr RTOS）**：处理时间关键任务，通过 RPC 向 MPU 暴露函数。

#### Arduino Router（基础设施）

在底层，通信由称为 **Arduino Router**（`arduino-router`）的后台 Linux 服务管理。

虽然 `Bridge` 库是您在代码中使用的内容，但 Router 是使其成为可能的流量控制器。它使用 MessagePack RPC 实现 **星型拓扑**网络。

**关键特性：**

- **多点通信：** 与简单的串行通信（通常是点对点）不同，Router 允许多个 Linux 进程同时与 MCU 通信（并相互通信）。

**Linux ↔ MCU：** 多个 Linux 进程可以同时与 MCU 交互（例如，一个 Python® 脚本读取传感器，而另一个 C++ 应用程序命令电机）。

**Linux ↔ Linux：** 您可以使用 Router 桥接在 MPU 上运行的不同应用程序。例如，Python 脚本可以暴露另一个 Python® 或 C++ 应用程序直接调用的 RPC 函数，允许服务在不涉及 MCU 的情况下交换数据。

- **服务发现：** 客户端（如您的 Python® 脚本或 MCU Sketch）"注册"它们想要暴露的函数。Router 保留这些函数的目录并将调用路由到正确的目的地。

**源代码：**

- [**Arduino Router Service**](https://github.com/arduino/arduino-router)
- [**Arduino_RouterBridge Library**](https://github.com/arduino-libraries/Arduino_RouterBridge/tree/main)

### SPI

UNO Q 支持 SPI 通信，允许开发板与其他 SPI 兼容设备之间进行数据传输。

UNO Q 用于 SPI 通信协议的引脚如下：

| 微控制器引脚 | Arduino 引脚映射 |
|--------------|------------------|
| PB9 | SS / D10 |
| PB15 | MOSI / D11 |
| PB14 | MISO / D12 |
| PB13 | SCK / D13 |

> 请参考用户手册的板卡引脚图部分，在板卡上定位这些引脚。

在 sketch 顶部包含 `SPI` 库以使用 SPI 通信协议。SPI 库提供 SPI 通信功能：

```cpp
#include <SPI.h>
```

在 `setup()` 函数中，初始化 SPI 库，定义并配置片选（SS）引脚：

```cpp
#define SS D10

void setup() {
  // 将片选引脚设置为输出
  pinMode(SS, OUTPUT);

  // 将 SS 引脚拉高以取消选择设备
  digitalWrite(SS, HIGH);

  // 初始化 SPI 通信
  SPI.begin();
}
```

要向 SPI 兼容设备传输数据，可以使用以下示例中的命令：

1. 在 Arduino App Lab 中创建新 App。

2. 将以下示例复制并粘贴到新 App 的 "sketch" 部分。

```cpp
#include <SPI.h>

#define SS D10

void setup() {
  // 将片选引脚设置为输出
  pinMode(SS, OUTPUT);

  // 将 SS 引脚拉高以取消选择设备
  digitalWrite(SS, HIGH);

  // 初始化 SPI 通信
  SPI.begin();
}

void loop() {
  // 替换为目标设备的地址
  byte address = 0x35;
  // 替换为要发送的值
  byte value = 0xFA;
  // 将 SS 引脚拉低以选择设备
  digitalWrite(SS, LOW);
  // 发送地址
  SPI.transfer(address);
  // 发送值
  SPI.transfer(value);
  // 将 SS 引脚拉高以取消选择设备
  digitalWrite(SS, HIGH);

  delay(2000);
}
```

### I2C

UNO Q 支持 I2C 通信，允许开发板与其他 I2C 兼容设备之间进行数据传输。UNO Q 用于 I2C 通信协议的引脚如下：

| 微控制器引脚 | Arduino 引脚映射 (Wire) | 微控制器引脚 | Arduino 引脚映射 (Wire1) |
|--------------|-------------------------|--------------|--------------------------|
| PB10 | SCL / D21 | PD12 | I2C4_SCL (Qwiic) |
| PB11 | SDA / D20 | PD13 | I2C4_SDA (Qwiic) |

> 请参考用户手册的板卡引脚图部分，在板卡上定位这些引脚。

要使用 I2C 通信，在 sketch 顶部包含 `Wire` 库。`Wire` 库提供 I2C 通信功能：

```cpp
#include <Wire.h>
```

在 `setup()` 函数中，初始化 I2C 库：

```cpp
// 初始化 I2C 通信
Wire.begin(); // UNO 风格接头上的 I2C (D20, D21)
// 或
Wire1.begin(); // Qwiic 连接器上的 I2C
```

要向 I2C 兼容设备传输数据，可以使用以下示例中的命令：

1. 在 Arduino App Lab 中创建新 App。

2. 将以下示例复制并粘贴到新 App 的 "sketch" 部分。

```cpp
#include <Wire.h>

void setup() {
  // 初始化 I2C 通信
  Wire.begin();
}

void loop() {
  // 替换为目标设备的 I2C 地址
  byte deviceAddress = 0x35;
  // 替换为适当的指令字节
  byte instruction = 0x00;
  // 替换为要发送的值
  byte value = 0xFA;
  // 开始向目标设备传输
  Wire.beginTransmission(deviceAddress);
  // 发送指令字节
  Wire.write(instruction);
  // 发送值
  Wire.write(value);
  // 结束传输
  Wire.endTransmission();

  delay(2000);
}
```

### Qwiic

Arduino UNO Q 配备板载 Qwiic 连接器，为连接 I²C 设备提供简单、免工具的解决方案。

Qwiic 连接器提供以下功能：
- 简化的 I2C 连接
- 即插即用的传感器和模块集成
- 无需焊接或面包板

### UART (串行通信)

UNO Q 支持 UART 串行通信，引脚映射如下：

| 微控制器引脚 | Arduino 引脚映射 | 引脚功能 |
|--------------|------------------|----------|
| PB7 | D0 / RX | GPIO / UART RX |
| PB6 | D1 / TX | GPIO / UART TX |

UART 通信可通过 Arduino 的 `Serial` 库使用。对于调试输出，可使用 `Monitor.begin()` 初始化串口监视器。

### CAN 总线

UNO Q 支持 CAN（控制器局域网）总线通信，引脚映射如下：

| 微控制器引脚 | Arduino 引脚映射 | 引脚功能 |
|--------------|------------------|----------|
| PA12 | D4 / FDCAN1_TX | GPIO / CAN 总线 TX |
| PA11 | D5 / FDCAN1_RX | GPIO / CAN 总线 RX |

CAN 总线适用于需要高可靠性和实时性能的应用场景，如汽车电子和工业自动化。
