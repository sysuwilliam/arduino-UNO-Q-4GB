# Arduino App Lab 入门指南

学习如何设置 Arduino App Lab 并在 UNO Q 开发板上启动 Apps。

**作者**：Karl Söderby  
**最后修订**：2026年2月28日

![Arduino App Lab](https://docs.arduino.cc/static/5fca8fb78876ee04bfdda00d7305c76b/a6d36/app-lab-hero.png)

通过 [Arduino App Lab](https://www.arduino.cc/en/uno-q/)，您可以在 [Arduino® UNO Q](https://store.arduino.cc/products/uno-q) 开发板上创建和部署 **Apps**，该开发板同时具有微控制器和运行 Linux 系统的微处理器。Arduino App Lab 设计为既可以在个人计算机（Windows、MacOS、Linux）上运行，也可以在 UNO Q 上运行，它已预装并自动更新。

在本指南中，我们将探讨：

- Arduino App Lab 的核心功能
- Apps 如何工作，以及它们的结构
- [Brick](https://docs.arduino.cc/software/app-lab/tutorials/bricks/) 是什么，以及它们如何在 Apps 中使用
- 如何在 UNO Q 开发板上启动 App

***Arduino App Lab 软件还提供了多个指南和示例，帮助您开始使用 UNO Q。***

---

## 硬件和软件要求

### 硬件

- 计算机（Windows/Mac/Linux）
- [Arduino® UNO Q 开发板](https://store.arduino.cc/products/uno-q)
- [USB-C® 数据线](https://store.arduino.cc/products/usb-cable2in1-type-c)

或者，将开发板用作[单板计算机（SBC）](https://docs.arduino.cc/tutorials/uno-q/single-board-computer/)时，需要以下设备：

- USB-C 转接器
- 显示器和 HDMI 数据线
- 键盘和鼠标

### 软件

- [Arduino App Lab](https://www.arduino.cc/en/uno-q/)

### 网络连接

当发布新更新时，开发板会自动更新。要下载和安装更新，需要网络连接。运行 Apps 不需要网络连接，除非使用特别需要网络的示例（例如连接到 Arduino Cloud、从外部 API 获取数据）。

## 安装和设置 Arduino App Lab

***各操作系统的具体说明可在下方找到。***

1. 导航到 [Arduino 软件页面](https://www.arduino.cc/en/software/#app-lab-section)，下载适用于您操作系统的 Arduino App Lab。

2. 下载并运行安装文件。

3. 启动 Arduino App Lab。

4. 将 UNO Q 开发板连接到计算机。

启动 Arduino App Lab 并连接 UNO Q 后，编辑器内会提示您配置开发板的 Wi-Fi®，以及为开发板设置名称和密码。

***您还会被提示登录 arduino.cc 账户。这是可选的，但您需要登录才能使用 Arduino App Lab 的某些功能。***

### MacOS

在 MacOS 上，从 Downloads 文件夹运行 `.dmg` 文件，并将其移动到 Applications 文件夹。

![MacOS installation](https://docs.arduino.cc/static/fd46aea323403dac5c7e8da2a8314577/a6d36/macos.png)

### Windows

在 Windows 上，从 Downloads 文件夹运行 `.exe` 文件，并完成安装。

![Windows Installation](https://docs.arduino.cc/static/db8b8833b1f5fa285f0b998bd6f6eff0/a6d36/windows.png)

### Linux

在 LinuxOS（基于 Ubuntu）上，解压下载的文件（`.tar.gz`）。导航到文件夹并运行应用程序。

![Linux Installation](https://docs.arduino.cc/static/47c9eaaffe357ae6ac8edcf03c770a84/a6d36/linux.png)

解压文件夹后，我们建议将其移动到例如 `$HOME/Desktop` 或 `$HOME/Applications`：

```
tar -xf ArduinoAppLab*.tar.gz 
mv ArduinoAppLab*/ ~/Desktop
```

**注意：** 您需要在计算机上安装 `libwebkit2gtk-4.1` 才能成功运行 Arduino App Lab。通过运行以下命令安装：

- **Debian / Ubuntu:**

```
sudo apt install libwebkit2gtk-4.1-0
```

- **Arch:**

```
sudo pacman -S webkit2gtk-4.1
```

***注意：为确保 UNO Q 开发板被 Arduino App Lab 正确检测，您的用户账户必须具有 USB 设备的特定写入权限。没有这些权限，开发板可能不会出现在应用程序中或无法连接。请参阅 UNO Q 用户手册中的 [Linux Host Setup](https://docs.arduino.cc/tutorials/uno-q/user-manual/#linux-host-setup-required-for-linux-users) 部分以安装必要的 `udev` 规则。***

## 开发板模式

开发板可以在**三种不同模式**下使用：

- **通过 USB（桌面模式）** - 开发板连接到个人计算机（Windows / Mac / Linux），并使用 Arduino App Lab 桌面应用程序进行编程。

- **通过本地 Wi-Fi® 网络（网络模式）<sup>1</sup>** - 使用此模式，通过 SSH<sup>2</sup> 在本地网络（通过 Wi-Fi®）上与开发板建立连接。

- **将开发板用作单板计算机（SBC 模式）** - 通过连接带有显示器、键盘和鼠标的 USB 转接器，UNO Q 可以用作 SBC。

***<sup>1</sup>网络模式仅在首次设置完成后才能使用。首次设置后，启动 Arduino App Lab 时，选择"Network"模式。***

***<sup>2</sup>连接到开发板的计算机需要连接到同一本地网络。***

### 通过 USB 的桌面模式（默认）

要使用**桌面模式**对开发板进行编程，请按照以下说明操作：

1. 使用 USB-C® 数据线将开发板连接到计算机。

2. 打开 Arduino App Lab，等待开发板出现<sup>3</sup>，然后选择 **USB** 选项。

![Desktop mode](https://docs.arduino.cc/static/ba053d477049d1b5200ae7497f51312e/a6d36/usb-mode.png)

3. 您已准备好在开发板上创建和运行 Apps ✅。

***<sup>3</sup>开发板需要一些时间启动。如果 LED 矩阵上的动画仍在运行，则表示仍在启动。***

### 网络模式（通过本地 Wi-Fi® 网络）

要使用**网络模式**对开发板进行编程，请按照以下说明操作：

1. 确保开发板已设置（通过 USB 连接开发板完成，参见上方说明）。

2. 打开 Arduino App Lab，等待开发板出现，然后选择 **network** 选项。

![Network mode](https://docs.arduino.cc/static/8adebd1065a40b9832542a27a8566267/a6d36/network-mode.png)

3. 您已准备好在开发板上创建和运行 Apps ✅。

***网络模式使得可以在不连接计算机的情况下在开发板上开发和运行 Apps。***

***注意：如果您更换 Wi-Fi® 网络，开发板需要再次通过 USB 连接，届时您将被提示连接到新网络。之后，开发板将自动连接到任何先前配置的网络。***

### SBC 模式（独立模式）

***此模式需要支持 HDMI 和外部供电的 USB 转接器。***

要使用 **SBC 模式**对开发板进行编程，请按照以下说明操作：

1. 将 USB 转接器连接到开发板的 USB-C® 端口。

2. 将显示器（通过 HDMI）、键盘和鼠标连接到 USB 转接器。

3. 使用 +5 VDC - 3 A 电源（例如手机充电器）为 USB 转接器供电。

4. 完整设置应如下所示：

![SBC setup](https://docs.arduino.cc/static/393aab342f7ccc542415e0446c555239/a6d36/sbc.png)

5. 等待开发板启动。启动完成后，显示器上将出现登录界面。

6. 使用 `arduino` 作为用户名登录，并为开发板创建自定义密码。

7. 登录后，Arduino App Lab 将自动启动并检查更新。更新完成后，您已准备好在开发板上创建和运行 Apps ✅。

![Arduino App Lab in SBC mode](https://docs.arduino.cc/static/5e4b0eda620f77aad494075bcaa48f4b/a6d36/app-lab-sbc.png)

## Arduino App Lab UI 概览

Arduino App Lab 设计为用于创建和启动 Apps 的编辑器和资源管理器。

![Arduino App Lab - Overview](https://docs.arduino.cc/static/a4690d2eb0020514844b359f5568a2ab/a6d36/app-overview.png)

- **My Apps** - 创建/复制的 Apps 出现在这里。点击每个 App 进行编辑和运行。

- **Examples** - Arduino® 团队制作的示例，包括音频分类、视频目标检测和 GPIO 控制。

- **Bricks** - 模块化代码构建块，用于轻松创建高级 Apps。

- **Learn** - 内置文档，用于学习 Arduino App Lab 功能。

---

### 什么是 Apps？

Arduino App Lab 围绕在 [UNO Q](https://store.arduino.cc/products/uno-q) 开发板上运行的 **Apps** 设计。UNO Q 是一块与众不同的 Arduino 开发板，具有能够运行 Debian OS（一种 Linux 发行版）的微处理器，以及运行 sketch 文件的微控制器。这些系统可以使用名为 [Bridge](#bridge-tool) 的工具进行通信。

Apps 是各种配置文件和应用程序文件的组合：

- 在 Linux 系统上运行的 **Python® 文件**
- 在微控制器上运行的 **sketch 文件**
- 或者 **Python® 文件和 sketch 文件两者**，相互通信（这是最常见的示例）

以下是 **Home Climate Monitoring** App 的示例，它从 Modulino® 记录数据并将其流式传输到托管 Web 应用程序的 Web 服务器。

![Home Climate Monitoring example App](https://docs.arduino.cc/static/ed2f8e8e9e528aa5618bf1c176dbb05b/a6d36/climate-monitoring.png)

Apps 可以直接从大量预制示例创建，也可以从头开始创建自己的 App。示例可在侧边菜单的 **"Examples"** 下访问，其中提供了应用内文档，说明如何连接任何额外的硬件、如何使用示例以及如何理解代码设置。所有示例都可以复制和修改<sup>4</sup>。

要创建自己的 App，请导航到 **"Apps"** 选项卡，然后点击 **"Create new app +"** 按钮。

***<sup>4</sup>要复制示例，请导航到示例，点击名称旁边的箭头，然后选择"Duplicate Example"。某些示例（如"Blinking LED from Arduino Cloud"）可能需要额外设置，例如添加 API 密钥。***

### 什么是 Bricks？

![Arduino App Lab - Bricks](https://docs.arduino.cc/static/e29e75a12e6fdef4e390b576a3275b10/a6d36/bricks.png)

[Bricks](https://docs.arduino.cc/software/app-lab/tutorials/bricks/) 与 Apps 不同，最好将其比作具有预打包功能的软件库，例如 AI 模型、Web 服务器或特定的 HTTP 请求，这些功能将使开发代码更容易。然而，Bricks 的独特之处在于它们不仅导入代码，它们被设计为与 Apps 并行启动。

有许多不同的类别，例如：

- 在特定图像集上训练的目标检测模型
- 用于连接外部 API 的预打包代码（例如天气预报）
- 托管 Web 应用程序的 Web 服务器

包含 AI 模型的 Bricks 作为 [docker](https://www.docker.com/) 容器与 App 一起部署，可能有多个容器同时运行。

Arduino App Lab 有许多可以直接安装和使用的 Bricks。这些可在侧边菜单的 **"Bricks"** 下找到，并经常更新。

***每个 Brick 的 API 文档可以在 Arduino App Lab 内的"Bricks"选项卡中访问。***

---

### App 文件

App 包含在一个具有多个子目录的目录中。典型的 App 看起来像这样：

![Edit Apps](https://docs.arduino.cc/static/aea108322ae2c16d22b48281d0bbf38f/a6d36/edit-apps.png)

有三个主要文件，每个文件包含在单独的文件夹中：

- `sketch.ino` - 包含**微控制器**的所有代码，使用 Arduino 编程语言（C++）编写

- `main.py` - 在微处理器上运行的 **Linux 系统**的主入口点，使用 Python® 编写

- `app.yaml` - 包含应用程序的元数据，如名称、描述和使用的 *Bricks*。此文件**无法编辑**

还有其他文件，如 `README.md`，存储 App 的文档，以及可能是特定示例一部分的附加文件。

## 创建和运行 Apps

![Arduino App Lab - Apps](https://docs.arduino.cc/static/2d9702a8671c9d2da32a315bc6bd7572/a6d36/app-examples.png)

要运行 App，您可以从头开始，或选择一个**现有示例**。对于首次使用的用户，我们建议使用现有示例以更好地理解结构。

### 运行示例 Apps

要运行示例 App，请按照以下步骤操作：

1. 从左侧侧边菜单的 **"Example"** 选项卡中选择要在 UNO Q 开发板上运行的示例。

2. 点击右上角的 **"Run"** 按钮。

3. 等待加载过程完成。

4. 启动完成后，您可以开始与 App 交互。

![Launching an app](https://docs.arduino.cc/static/59094559f71c46cf5b5615ab43ce4c81/a6d36/launch-app.png)

每个官方示例都在示例 App 内的 `readme.md` 文件中提供了详细文档。

***请注意，每次启动 App 时，都会编译 sketch，并启动 Python® 应用程序。根据应用程序的复杂性，此过程可能需要长达一分钟。***

### 如何知道我的 App 正在运行？

一旦启动 App，App 的状态将在 **Console** 中显示。有三个可用的选项卡：

- **Start-up** - 输出启动过程的日志。这里您将找到有关微控制器编译和 Python® 应用程序在 Linux 系统上部署的信息。

- **Main (Python®)** - 查看 Python® 应用程序的日志（`print()`）

- **Sketch (Microcontroller)** – 查看使用 `Monitor.print()` 从 sketch 发送的数据

**注意：** `Serial.println()` 通过 UART 输出数据，不会显示在 App Lab Console 中。要在 Console 中显示消息，请使用 `Monitor.print()`。在 [From Serial to Monitor](https://docs.arduino.cc/tutorials/uno-q/user-manual/#from-serial-to-monitor) 中了解更多信息

![App logs](https://docs.arduino.cc/static/e05847b6a42ea7462c2beaeb971e156f/a6d36/app-logs.png)

***请注意，虽然 App 可能成功启动，但这并不意味着它工作正常。Python® 脚本可能有问题，这可以在 Python 日志中看到。但是，如果在 sketch 编译期间发生错误，启动将被中止。***

### 创建新 App

要创建新 App，请在主页上点击 **"Create New App"**。

从头开始创建新 App 时，`sketch.ino` 和 `main.py` 将自动创建启动应用程序所需的最少代码，而 Bricks 需要手动导入。

要添加 Brick，请点击 **"Add Bricks"** 按钮，然后选择您想使用的 Brick。现在可以在您的 `main.py` 文件中使用该 Brick。

### 复制现有示例

除了从头开始创建新 App 外，示例也可以被**复制**。这是通过在 **"Examples"** 部分选择可用示例并复制它来完成的：

![Duplicating an example](https://docs.arduino.cc/static/8af53032423af5d2dd0ae95fd92ae526/a6d36/cloud-blink-duplicate.png)

## Apps 如何运作

一次只能有一个 App 在开发板上运行，但 App 可能有多个 Bricks 在后台运行。Bricks 作为独立进程部署在开发板上运行，App 可以使用特定 API 与其交互。

例如，一个 App 可能同时运行以下内容：

- AI 模型，用于分类传入数据（例如音频、摄像头、传感器数据）
- Web 服务器，显示数据
- 连接到 Web API 的 Brick

这些进程并行运行，这使得 Apps 成为创建复杂项目时非常通用的组件。

### App 运行

创建 App 时，重要的是始终导入并使用 `App` 类，特别是 `run()` 函数。

```
# Launches the App, along with any imported Bricks
App.run()
```

重要的是将 `App.run()` 放在 `main.py` 文件的末尾，因为这将启动任何导入的 Bricks 和工具（如 Bridge）。放置在此之后的任何代码将无法正常工作。

### Bridge 工具

开发 Apps 时最重要的工具之一是 **Bridge** 工具。Bridge 工具使得可以通过易于使用的 API 在微控制器和 Linux 系统之间进行通信，具有三个核心功能：

- **Provide** - *提供*一个可以被*调用*的服务
- **Call** - 我们*调用*一个被*提供*的服务
- **Notify** - 我们使用一些参数向另一端*通知*（单向数据推送）

简单来说，如果您想从微控制器向 Linux 系统发送数据，我们将实现一个提供者函数，然后可以从 Linux 系统调用它，或使用 notify 函数将数据从一个系统推送到另一个系统。

这是一个非常快速的示例，演示从 sketch 向 Python® 应用程序发送数据。

**Sketch:**

```
int data = 1;
Bridge.notify("python_function", data);
```

**Python®:**

```
Bridge.provide("python_function", python_function)

def python_function(data: int)
    print(data)
```

## 总结

Arduino App Lab 是一个多功能的编辑器，用于在 UNO Q 开发板上编程 Python 和 Sketch 应用程序。它包括大量快速入门示例，以及多个使开发体验更轻松的 Bricks。

在本指南中，我们涵盖了编辑器的一些最重要方面，例如：

- 如何首次安装和设置开发板
- *Apps* 是什么，它们由什么组成
- *Bricks* 是什么，以及它们如何运作
- *Bridge* 是什么，以及它如何使 MPU 和 MCU 通过 RPC 通信

更多 Arduino App Lab 文档可在以下位置找到：

- [Arduino App Lab Documentation](https://docs.arduino.cc/software/app-lab/)

您还可以访问 [Arduino® UNO Q](https://docs.arduino.cc/hardware/uno-q) 硬件页面了解开发板详情。
