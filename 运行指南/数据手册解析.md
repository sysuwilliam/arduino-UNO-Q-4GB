## 一、产品概述

### 产品定义

**Arduino UNO Q** 是一款单板计算机，结合了：

| 组件 | 说明 |
|------|------|
| **微处理器 (MPU)** | Qualcomm® Dragonwing™ QRB2210，四核 Arm® Cortex®-A53，运行 Debian Linux OS |
| **微控制器 (MCU)** | STMicroelectronics STM32U585，Arm® Cortex®-M33，运行 Arduino Core on Zephyr OS |

### 核心特性

- **双架构设计**：Linux 系统与微控制器通过 **Bridge**（Arduino 的 RPC 远程过程调用库）进行通信
- **存储配置**：
  - eMMC 存储：16 GB 或 32 GB 选项
  - LPDDR4X SDRAM：2 GB 或 4 GB 选项
- **无线连接**：双频 Wi-Fi® 5 + Bluetooth® 5.1
- **USB-C 接口**：支持 Power Delivery 输入和视频输出
- **Arduino 兼容**：标准 UNO 引脚，支持 Shield 扩展

### 开发环境

- **Arduino App Lab**：统一开发环境，支持：
  - Arduino Sketch
  - Linux 应用程序
  - AI 模型集成
- 可在板上直接运行（SBC 模式）或从连接的 PC 运行

## 五、处理器架构

### 5.1 MPU (应用处理器)

**Qualcomm® Dragonwing™ QRB2210**

| 特性 | 说明 |
|------|------|

**STMicroelectronics® STM32U585**

| 特性 | 说明 |
|------|------|

## 六、处理器间通信

### Bridge RPC 通信

**Arduino Bridge** 是软件实现的远程过程调用（RPC）层，在 Linux 端和 MCU 端都有实现。

#### Bridge 功能

| 功能 | 说明 |
|------|------|
| **服务暴露** | 任一处理器可以暴露服务供另一处理器调用 |
| **异步通知** | 支持单向通知事件 |
| **消息路由** | 管理处理器间的消息路由 |
| **多传输支持** | 支持多种物理传输方式 |
| **类型安全** | API 提供类型安全的函数调用 |

#### 通信场景

#### 硬件指示信号（高级应用）

如果载板或外部逻辑需要硬件指示：

1. 固件可指定 1.8V MPU GPIO（JMISC）或 JCTL GPIO 作为就绪/唤醒输出
2. 通过电平兼容电路（电平转换器或开漏配置+上拉电阻）连接到 MCU GPIO
3. 或者，传输活动（USB CDC、UART、SPI）可作为 MCU 睡眠模式的唤醒源

> ⚠️ **电压兼容性：** MPU GPIO 信号工作在 1.8V 域，连接到 MCU（3.3V 域）时需使用电平转换器或开漏配置。

## 七、硬件加速

### 7.1 图形加速

#### 图形 API 支持

| 图形 API | 驱动 | 硬件支持 | 驱动版本 | 设备名称 |

| Desktop OpenGL | freedreno | - | 3.1 | FD702 |
| OpenGL ES | freedreno | 3.1 | 3.1 | FD702 |
| Vulkan | turnip | 1.1 | 1.0.318 | Turnip Adreno 702 |
| OpenCL | Mesa | 2.0 | 2.0 | - |

#### GPU 规格

| 参数 | 规格 |
|------|------|
| 时钟频率 | 845 MHz |
| 内存架构 | 统一架构（与系统 RAM 共享） |
| 可用显存 | 1740 MB |
| 内存寻址 | 64-bit |
| 直接渲染 | 是 |
| 最大 2D 纹理 | 16384 × 16384 像素 |
| 最大 3D 纹理 | 2048³ 体素 |
| 最大 Cube Map | 16384 × 16384 像素 |

### 7.2 视频加速

通过 V4L2 (Video4Linux2) API 访问 `/dev/video0` 和 `/dev/video1` 设备。

#### 视频编解码支持

| 编解码 | 编码 | 解码 | GStreamer 元素 |
|--------|------|------|----------------|
| **H.264 (AVC)** | ✓ | ✓ | v4l2h264enc / v4l2h264dec |
| **H.265 (HEVC)** | ✓ | ✓ | v4l2h265enc / v4l2h265dec |
| **VP9** | ✗ | ✓ | v4l2vp9dec |

> **分辨率支持：** 最高 1920×1080 (Full HD)，包括 720p (1280×720)

#### GStreamer 示例

**H.264 解码：**

```bash
gst-launch-1.0 filesrc location=videos/xxxxx.mp4 \
! qtdemux name=demux demux.video_0 ! queue ! h264parse ! v4l2h264dec \
! videoconvert ! autovideosink
```

**H.265 解码：**

```bash
gst-launch-1.0 filesrc location=videos/xxxxx.mp4 \
! qtdemux name=demux demux.video_0 ! queue ! h265parse ! v4l2h265dec \
! videoconvert ! autovideosink
```

**H.264 编码：**

```bash
gst-launch-1.0 videotestsrc num-buffers=30 \
! video/x-raw,width=1280,height=720,framerate=30/1 \
! v4l2h264enc ! h264parse ! mp4mux ! filesink location=/tmp/output.mp4
```

**并发编码解码：**

```bash
gst-launch-1.0 -v videotestsrc num-buffers=1000 \
! video/x-raw,format=NV12,width=1280,height=720,framerate=30/1 \
! v4l2h264enc capture-io-mode=4 output-io-mode=2 ! h264parse \
! v4l2h264dec capture-io-mode=4 output-io-mode=2 ! videoconvert \
! autovideosink
```

### 7.3 OpenCL 支持

- 并行处理任务
- 科学计算

### 8.2 JANALOG (A3) - 模拟 I/O

**14-pin，3.3V 逻辑**

| 引脚 | 标识 | MCU 引脚 | 功能 | 备注 |
|------|------|----------|------|------|
| 1 | BOOT | MCU_BOOT0 | 启动选择 | - |
| 2 | IOREF | PWR_3P3V | I/O 电压参考 | 仅输出，不可反向供电 |
| 3 | RESET | MCU_NRST | MCU 复位 | - |
| 4 | +3V3 OUT | PWR_3P3V | 3.3V 供电 | - |

| 6 | GND | GND | Ground | - |
| 7 | GND | GND | Ground | - |
| 8 | VIN IN | DC_IN | 7-24V 输入 | 仅电源 |
| 9 | A0/D14 | PA4 | ADC, DAC0, TIM2_CH1 | **非 5V 容忍** |

### 8.3 Qwiic (A4) - I²C 接口

**4-pin，3.3V 逻辑**

| 引脚 | 标识 | MCU 引脚 | 功能 |
|------|------|----------|------|
| 1 | GND | - | Ground |
| 2 | +3V3 OUT | - | 3.3V 供电 |
| 3 | SDA | PD13 (I2C4_SDA) | I²C 数据 |
| 4 | SCL | PD12 (I2C4_SCL) | I²C 时钟 |

> 兼容 Qwiic 生态系统和 Modulino® 节点

### 8.4 JSPI (A5) - 专用 SPI

**6-pin，3.3V 逻辑 + 5V VBUS**

## 十一、设备操作指南

### 11.1 快速开始 - Arduino App Lab

#### App 组成

一个 App 可包含：

1. **Python 程序**：运行在 Linux 系统（MPU）
2. **Arduino Sketch**：运行在微控制器（MCU）
3. **Brick**：预打包服务（AI 模型、Web 服务器、API 客户端）

#### 使用流程

1. 安装 Arduino App Lab，启动并连接 UNO Q
2. 板子会自动检查更新
3. 首次设置时，提供设备名称、密码和 Wi-Fi 凭据
4. 在 Examples 区域导航到示例 App，点击 Run
5. 在 Console 标签页监控 App 状态

#### 操作模式

| 模式 | 连接方式 | 说明 |
|------|----------|------|
| **PC-Hosted** | USB-C 数据线 | 需要数据连接，首次设置必需 |
| **SBC** | 仅电源 + 网络 | 无需 USB 数据，使用 Network target |

### 11.2 Bricks（模块化构建块）

Bricks 是 Arduino App Lab 中的模块化构建块，提供：

- AI 模型（对象分类、关键词检测）
- Web UI 或 REST API 服务
- 外部数据源集成

#### 工作流程

1. 在 Arduino App Lab 中创建 App
2. 选择需要的 Brick
3. 添加 Python 代码和/或 Arduino Sketch
4. 在 `main.py` 中导入并初始化 Brick
5. 点击 Run 部署

### 11.3 Hello World 示例

1. 打开 Arduino App Lab，进入 Examples 区域
2. 打开 **Blink LED** 示例
3. 点击 Run，等待上传完成
4. 观察内置 RGB LED 红色通道每秒闪烁

> LED 由 STM32U585 微控制器通过 Arduino Sketch 驱动。

### 11.4 检查 App 运行状态

Console 标签页包含三个子标签：

| 标签 | 内容 |
|------|------|
| **Start-up** | 启动序列日志，包括 MCU 编译和 Linux 部署 |
| **Main (Python)** | Python 应用输出（print()） |
| **Sketch (Microcontroller)** | Arduino Sketch 串口输出（Serial.println()） |

### 11.5 电源按钮

| 操作 | 效果 |
|------|------|
| **长按 5+ 秒** | 重启 Linux 部分 |

> 板子上电时会自动启动，无需按下电源按钮。
