# Arduino UNO Q Debian Linux 基础指南

全面了解 UNO Q 的 Debian 环境，快速上手使用。
 
**最后修订**: 2026年3月16日

---

## 概述

Arduino UNO Q 在其 Qualcomm® QRB2210 微处理器（MPU）上运行完整的 Debian Linux 操作系统，与 Arduino 微控制器（MCU）并行提供一个完整的计算环境。

本指南介绍 Debian 系统，包括如何访问、导航文件系统、管理权限、安装软件、使用 USB 外设，以及值得牢记的最佳实践。

了解 Debian 系统可以让你充分利用 Linux 的强大功能进行高级项目、系统管理和超越 Arduino sketches 的开发任务，从而充分发挥 UNO Q 的潜力。

---

## 什么是 Debian？

Debian 是一个以稳定性、安全性和丰富的软件包生态系统而闻名的免费开源 Linux 发行版。在 UNO Q 上，Debian 提供完整的 Linux 环境和标准 Unix 工具，允许你同时运行多个应用程序，并通过 `apt` 包管理器访问数千个软件包。

该系统内置支持网络服务，如 SSH、Web 服务器和各种网络协议，以及预装的开发工具，包括编译器、解释器和开发库。

### Debian 如何在 UNO Q 上工作

UNO Q 采用混合架构，两个不同的处理环境协同工作。Qualcomm® QRB2210 微处理器运行 Debian Linux，处理高级计算任务，包括管理 Wi-Fi® 和蓝牙® 等网络功能、运行 Python® 脚本和 Linux 应用程序，以及控制系统资源和外设。

与之并行的是运行 Zephyr RTOS 的 STM32U585 微控制器，它运行 Arduino sketches、处理实时操作、控制 GPIO 引脚和硬件接口，并通过 **Bridge** 机制与 MPU 通信。

**Bridge 机制**是使两个系统能够不间断通信的关键元素。它允许你在 Debian 上运行的 Python® 脚本与 MCU 上运行的 Arduino 代码进行交互，创建一个灵活的混合开发环境，同时利用 Linux 的计算能力和 Arduino 的实时能力。

Python® 是 UNO Q 上 MPU 端开发的官方支持语言。随着开发的继续，将提供更多编程语言和 MPU/MCU 通信的替代方法，为将自定义应用程序与硬件集成提供更多选择。

> **注意**: 如果你想在深入了解本 Debian 指南之前了解 UNO Q 的总体情况，请参阅 [UNO Q 用户手册](https://docs.arduino.cc/tutorials/uno-q/user-manual/) 以熟悉开发板。

## 访问开发板 Shell

有四种方法可以访问 UNO Q 上的 Debian shell，每种方法适用于不同的场景和工作流程。

### 通过 USB-C® (ADB)

Android Debug Bridge (ADB) 通过 USB 连接提供直接的 shell 访问，非常适合在开发板连接到计算机时快速访问，无需任何网络设置。

使用此方法需要 USB-C® 线缆和计算机上安装的 ADB 工具。连接后，可以通过运行 `adb devices` 验证开发板，该命令会列出所有已连接的设备。

```bash
adb devices
```

然后运行 `adb shell` 进入开发板的 shell 环境。此方法在初始设置期间或网络连接不可用时非常有用。

```bash
adb shell
```

### 通过 SSH (网络)

Secure Shell (SSH) 通过本地网络提供远程访问，允许你从同一网络上的任何地方连接到开发板，无需物理线缆。

此方法要求开发板已连接 Wi-Fi® 且 SSH 已启用（在 Arduino App Lab 完成首次设置后即启用）。**你的计算机和开发板必须在同一网络上才能工作。**

要连接，在计算机上打开终端并运行：

```bash
ssh arduino@<boardname>.local
```

将 `<boardname>` 替换为你的实际开发板名称。首次连接时，系统会要求你输入 `yes` 验证连接。确认后，输入开发板密码获得完整的 shell 访问权限。

示例：
```bash
ssh arduino@unoqtestbench.local
```

如果遇到连接问题，特别是在某些网络上的 mDNS 问题，可以直接连接到开发板的 IP 地址而不是 `.local` 主机名。通过运行以下命令查找 IP 地址：

```bash
hostname -I
```

> **注意**: 有关详细的 SSH 设置、使用 SCP 进行文件传输和全面的故障排除，请参阅专门的 [UNO Q 通过 Secure Shell (SSH) 教程](https://docs.arduino.cc/tutorials/uno-q/ssh/)。

### 通过 SBC 模式

单板计算机模式将你的 UNO Q 变成具有图形用户界面的独立桌面计算机。使用此模式，你需要一个 [**支持外部供电的 USB-C® 转接器**](https://store.arduino.cc/products/usb-c-to-hdmi-multiport-adapter-with-ethernet-and-usb-hub)、一个 HDMI 显示器以及 USB 键盘和鼠标。

> **重要**: 你可以使用任何支持外部供电功能的 USB-C 转接器，但 [Apple](https://www.apple.com/shop/product/mw5m3am/a/usb-c-digital-av-multiport-adapter) 的转接器除外。

将所有外设通过 USB-C® 转接器连接，启动系统，然后使用你的凭据登录。登录后，你可以直接从桌面环境打开终端应用程序，在图形界面旁边获得 shell 访问权限。

此方法非常适合将 UNO Q 用作工作站，允许你浏览网页、图形化编辑文件，并从同一设备访问终端。

> **注意**: 有关完整的单板计算机模式教程，请参阅专门的 [UNO Q 单板计算机教程](https://docs.arduino.cc/tutorials/uno-q/single-board-computer/)。

### 通过硬件调试 UART

对于低级调试和早期启动诊断，开发板通过 JCTL 连接器提供了一个专用的硬件调试 UART 接口，工作在 1.8 V 逻辑电平（115200 波特率）。

此接口直接连接到 SoC 的主控制台（TTY），允许你在 SSH 或 ADB 等网络服务可用之前监控启动和内核日志、排除系统问题或访问 shell 环境。你可以使用 Linux 凭据登录，通过 shell 与系统交互。

> **注意**: 此方法需要 1.8V USB 转 TTL 转换器，适用于高级用户。有关完整的设置说明和技术规格，请参阅用户手册中的 [硬件调试 UART 接口](https://docs.arduino.cc/tutorials/uno-q/user-manual/#hardware-debug-uart-interface) 部分。

## 系统导航

### 理解文件系统

Debian 以层次目录结构组织文件，根目录 `/` 位于顶层，所有其他目录在其下方。

对于大多数 UNO Q 项目，你主要在 `/home/arduino/`（你的默认用户目录）中工作。

在你的主目录中，`ArduinoApps/` 文件夹很重要，因为它包含你使用 Arduino App Lab 创建的所有 Apps，每个都存储在自己的子目录中。

```
/                       # 根目录（顶层）
├── home/
│   └── arduino/        # 你的工作目录
│       └── ArduinoApps/  # 你的 Arduino App Lab 项目
├── tmp/                # 临时文件（重启时清除）
└── mnt/                # USB 驱动器的挂载点
```

在导航系统时，你大部分时间会花在 `/home/arduino/`。`/tmp/` 目录用于临时文件，而当你挂载 USB 存储设备时 `/mnt/` 变得相关。

### 导航命令

`cd` 命令是导航目录结构的常用工具。运行 `cd ~` 直接进入你的主目录，而 `cd /home/arduino/Documents` 导航到特定路径。

```bash
cd ~
cd /home/arduino/Documents
```

要向上移动一级目录，使用 `cd ..`，要返回上一个目录，使用 `cd -`。你的命令提示符会更改以显示当前目录位置，帮助你跟踪在文件系统中的位置。

```bash
cd ..
cd -
```

要查看目录内容，使用 `ls` 命令。单独运行 `ls` 显示文件和目录的基本列表。`ls -lh` 命令提供详细列表，包括权限、所有权和人类可读的文件大小。

```bash
ls
ls -lh
```

添加 `-a` 标志（`ls -a`）会显示以点开头的隐藏文件，你可以组合多个选项如 `ls -lah` 来查看所有文件的完整详情。

如果你需要确切知道在目录结构中的位置，`pwd` 命令会打印当前工作目录。例如，运行 `pwd` 可能显示 `/home/arduino/Documents`，确认你在文件系统中的当前位置。

```bash
pwd
```

### 文件操作

使用 `mkdir` 命令创建目录很简单。运行 `mkdir my_project` 在当前位置创建新目录。

```bash
mkdir my_project
```

另一方面，`mkdir -p projects/arduino/sketches` 一次创建所有嵌套目录，包括任何尚不存在的中间目录。这在设置复杂项目结构时很有用。

```bash
mkdir -p projects/arduino/sketches
```

`cp` 命令处理文件复制操作。要复制单个文件，使用 `cp source.txt destination.txt`，它会创建一个具有新名称的副本。

```bash
cp source.txt destination.txt
```

处理目录时，需要递归标志：`cp -r source_folder/ destination_folder/` 复制整个目录及其所有内容。这在需要备份或复制项目文件夹时必不可少。

```bash
cp -r source_folder/ destination_folder/
```

移动和重命名文件都使用 `mv` 命令。区别在于你如何使用它：`mv oldname.txt newname.txt` 重命名当前目录中的文件。

```bash
mv oldname.txt newname.txt
```

而 `mv file.txt /home/arduino/Documents/` 将文件移动到不同位置。与复制不同，移动不会创建副本，它重新定位原始文件。

```bash
mv file.txt /home/arduino/Documents/
```

删除文件需要谨慎。`rm file.txt` 命令永久删除文件，而 `rm -r folder/` 删除整个目录及其内容。

```bash
rm file.txt
rm -r folder/
```

最有效但也最危险的变体是 `rm -rf folder/`，它会强制删除所有内容而不要求确认。此命令应极其谨慎使用，因为删除的文件无法恢复。

```bash
rm -rf folder/
```

> **警告**: `rm -rf` 命令永久删除文件而不确认，无法撤销。在运行此命令之前始终仔细检查路径，以避免意外数据丢失。

## 权限和超级用户访问

### 理解 Linux 权限

Debian 中的每个文件和目录都有相关的权限，控制谁可以**读取**、**写入**或**执行**它。

- **读取**权限允许查看文件内容或列出目录内容
- **写入**权限允许修改或删除文件
- **执行**权限允许将文件作为程序运行或访问目录

这些权限分配给三类用户：
- 创建文件的所有者
- 与文件同组的用户
- 系统上的所有其他用户

此权限系统提供对文件的受控访问和系统安全性。

### 使用 sudo (超级用户执行)

许多系统操作需要管理员权限，以防止对关键系统文件进行意外修改。`sudo` 命令允许你*临时*以超级用户权限运行命令。

例如，安装软件通常需要提升权限，所以你会使用 `sudo apt install package-name`。

```bash
sudo apt install package-name
```

同样，编辑系统配置文件如 `sudo nano /etc/network/interfaces` 需要超级用户访问。

```bash
sudo nano /etc/network/interfaces
```

当使用 `sudo systemctl restart networking` 等命令控制系统服务，或访问受保护目录如 `sudo ls /root` 时，系统会在命令运行前提示你输入密码。此安全措施确保关键系统更改是有意且经过身份验证的。

```bash
sudo systemctl restart networking
sudo ls /root
```

### 更改文件权限

`chmod` 命令修改文件和目录权限。当你需要使脚本可执行时，运行 `chmod +x script.sh` 添加执行权限，允许你用 `./script.sh` 运行脚本。

```bash
chmod +x script.sh
```

要更多控制，可以使用数字表示法如 `chmod 755 script.sh`，它授予所有者完整的读、写和执行权限，而其他人只能读取和执行文件。

```bash
chmod 755 script.sh
```

当你需要将权限应用于整个目录及其内容时，递归标志 `chmod -R 755 folder/` 确保文件夹内的所有文件都具有相同的权限。

```bash
chmod -R 755 folder/
```

你会遇到的常见权限代码包括：
- `755` - 给所有者完全控制，其他人可读取和执行
- `644` - 允许所有者读写，其他人只能读取
- `777` - 授予所有人完全访问权限，由于安全影响应谨慎使用

### 更改所有权

可以使用 `chown` 命令修改文件所有权，这在文件以错误所有权创建或需要在用户之间转移时很有用。

```bash
sudo chown arduino file.txt
```

运行 `sudo chown arduino file.txt` 将文件的所有者更改为 arduino 用户。你可以同时使用 `sudo chown arduino:arduino file.txt` 更改所有者和组。

```bash
sudo chown arduino:arduino file.txt
```

对于整个目录结构，递归选项 `sudo chown -R arduino:arduino /home/arduino/project/` 确保项目文件夹内的所有文件都具有相应的所有权。

```bash
sudo chown -R arduino:arduino /home/arduino/project/
```

## 文件管理

### 查看文件内容

当你需要查看文本文件内容时，Debian 提供了适合不同场景的多种工具。

最简单的是 `cat file.txt`，它直接将整个文件打印到终端。这对短文件效果很好，但对较长文档可能会让人眼花缭乱。

```bash
cat file.txt
```

对于较大的文件，`less file.txt` 命令打开一个可滚动的查看器，你可以使用箭头键导航，用 `/` 搜索文本，按 `q` 退出。它对如何阅读文件提供了更多控制。

```bash
less file.txt
```

有时你只需要看文件的开头或结尾。`head file.txt` 命令显示前 10 行，而 `tail file.txt` 显示最后 10 行。

```bash
head file.txt
tail file.txt
```

你可以使用 `head -n 20 file.txt` 等命令自定义显示的行数。

```bash
head -n 20 file.txt
```

`tail -f` 命令很有用，因为它持续监控文件并显示新添加的行。这非常适合查看日志文件或实时监控输出。

```bash
tail -f mylogfile.txt
```

要在 UNO Q 上监控系统日志，可以使用 `journalctl` 命令，它会实时显示系统日志：

```bash
sudo journalctl -f
```

### 使用 Nano 编辑文件

Nano 是一个用户友好的终端文本编辑器，预装在 UNO Q 上，是命令行编辑配置文件和脚本的选择。

要编辑文件，运行 `nano myfile.txt`，它会打开现有文件或创建新文件（如果不存在）。

```bash
nano myfile.txt
```

编辑需要提升权限的系统文件时，在命令前加上 sudo，如 `sudo nano /etc/hostname`。

```bash
sudo nano /etc/hostname
```

进入 Nano 后，使用箭头键在文本中移动，你会看到底部显示可用命令的有用菜单。

要保存更改，按 **CTRL + O**（写入），确认文件名，然后按 Enter。其他有用的命令包括：
- **CTRL + X** - 退出 nano
- **CTRL + K** - 剪切当前行
- **CTRL + U** - 在别处粘贴
- **CTRL + W** - 搜索文本
- **CTRL + G** - 访问完整帮助菜单

当你按 `CTRL + O` 保存时，Nano 会将你的更改写入存储，使其永久生效。

### 搜索文件和内容

`find` 命令帮助根据各种条件定位文件。要在主目录中查找所有 Python 文件，运行 `find /home/arduino -name "*.py"`，它会递归搜索所有子目录。

```bash
find /home/arduino -name "*.py"
```

你也可以根据修改时间搜索。例如，`find /home/arduino -mtime -7` 查找过去七天修改过的所有文件。这在大型目录结构中定位最近更改的文件很有用。

```bash
find /home/arduino -mtime -7
```

对于在文件内容而非文件名中搜索，`grep` 命令很有用。运行 `grep "Arduino" file.txt` 在特定文件中搜索单词 "Arduino" 并显示匹配的行。

```bash
grep "Arduino" file.txt
```

要搜索整个目录结构，使用 `grep -r "TODO" /home/arduino/`，它递归搜索所有文件中的文本 "TODO"。

```bash
grep -r "TODO" /home/arduino/
```

不区分大小写的选项 `grep -i "arduino" file.txt` 会匹配 "Arduino"、"arduino" 和 "ARDUINO"，使搜索更灵活。

```bash
grep -i "arduino" file.txt
```

## 软件包管理

Debian 使用高级软件包工具（通常称为 `apt`）来管理软件安装、更新和删除。此系统自动处理依赖关系，确保当你安装程序时，所有需要的支持库和工具也会安装。

### 更新软件包列表

在安装新软件或升级系统之前，必须更新软件包列表以确保使用最新版本。

运行 `sudo apt update` 从 Debian 仓库下载最新的软件包信息。你会看到列出正在检查的各种仓库，然后是可用升级的摘要。

```bash
sudo apt update
```

输出会显示类似这样的内容：

```
Hit:1 http://deb.debian.org/debian trixie-backports InRelease
Hit:2 http://deb.debian.org/debian trixie InRelease
Hit:3 http://deb.debian.org/debian trixie-updates InRelease
Hit:4 http://deb.debian.org/debian-security trixie-security InRelease
Hit:5 https://apt-repo.arduino.cc stable InRelease
58 packages can be upgraded. Run 'apt list --upgradable' to see them.
```

此过程不安装任何东西。它刷新系统对可用软件版本的了解。`Hit` 消息表示你的软件包列表已与仓库同步。如果有软件包可以升级，系统会通知你数量。

### 升级系统

更新软件包列表后，你可以将已安装的软件包升级到最新版本。`sudo apt upgrade` 命令更新所有已安装的软件包，同时对依赖关系保持保守，提供系统稳定性。

```bash
sudo apt upgrade
```

对于可能更改软件包依赖关系或安装升级所需新软件包的更全面更新，使用 `sudo apt full-upgrade`。此命令处理复杂的升级场景，比标准升级更彻底。

```bash
sudo apt full-upgrade
```

定期更新有助于获得最新的安全补丁和错误修复，保持你的 UNO Q 安全稳定。定期运行 `sudo apt update && sudo apt upgrade` 是良好习惯。

> **注意**: 对于主要系统更新或操作系统版本升级，建议使用专门教程中描述的镜像刷写程序：[UNO Q 镜像刷写教程](https://docs.arduino.cc/tutorials/uno-q/update-image/)。

### 安装软件

使用 apt 安装软件很简单。要安装单个软件包，使用 `sudo apt install package-name`，它会下载软件包及其所有依赖项，然后自动安装所有内容。

```bash
sudo apt install package-name
```

你可以在单个命令中安装多个软件包，用空格分隔，如 `sudo apt install python3-pip git curl`。安装期间，apt 会显示将要安装的内容，并在继续前要求确认。

```bash
sudo apt install python3-pip git curl
```

例如，如果要安装 Vim 文本编辑器，运行 `sudo apt install vim` 会下载 Vim 及任何所需的库，然后配置所有内容使其立即可用。

```bash
sudo apt install vim
```

软件包管理器在后台处理所有复杂性，使软件安装比手动编译和配置更简单。如果软件包已经安装，它会提示软件包的可用性和已安装版本。

### 搜索和管理软件包

当你不确定确切的软件包名称时，`apt search keyword` 命令帮助查找与主题相关的软件包。

例如，`apt search python3` 会列出所有与 Python 3 相关的可用软件包，每个都有简要描述。这对于探索特定任务或编程语言可用的工具很有用。

```bash
apt search keyword
apt search python3
```

删除软件也很简单。`sudo apt remove package-name` 卸载软件包但保留你可能自定义的任何配置文件。

```bash
sudo apt remove package-name
```

如果你想完全删除软件包包括其配置文件，改用 `sudo apt purge package-name`。随着时间的推移，你的系统可能会从以前安装的软件包积累未使用的依赖项。

```bash
sudo apt purge package-name
```

运行 `sudo apt autoremove` 自动清理它们，删除不再需要的软件包以释放磁盘空间。

```bash
sudo apt autoremove
```

## USB 和外设访问

### 检测 USB 设备

当你通过 SBC 模式下的 USB-C 转接器将 USB 设备连接到 UNO Q 时，Debian 会检测并注册它。

`lsusb` 命令提供所有已连接 USB 设备的快速概览，显示制造商 ID、产品 ID 和设备描述。你可能会看到键盘、鼠标、摄像头或其他外设的列表。

```bash
lsusb
```

输出显示已连接设备的格式如：

```
Bus 002 Device 001: ID 1d6b:0003 Linux Foundation 3.0 root hub
Bus 001 Device 003: ID 046d:c52b Logitech, Inc. USB Keyboard
Bus 001 Device 002: ID 093a:2510 USB Camera
```

有关 USB 设备的更详细信息，包括电源要求、传输速度和设备能力，可以使用 `lsusb -v` 获取详细输出，或 `lsusb -t` 查看显示设备如何通过集线器和端口连接的 USB 设备树。

```bash
lsusb -v
lsusb -t
```

### 使用 USB 存储设备

当你在 SBC 模式下将 USB 闪存驱动器或外部硬盘连接到桌面环境时，系统通常会在 `/media/arduino/` 自动挂载它，使其在文件管理器中可访问。

但是，从命令行或无头模式工作时，你需要手动挂载驱动器。

首先，通过运行 `lsblk` 识别你的 USB 驱动器，它会列出所有块设备及其分区。你会看到显示设备名称如 `/dev/sdb1` 以及其大小和挂载点的输出。

```bash
lsblk
```

要手动挂载 USB 驱动器，用 `sudo mkdir -p /mnt/usb` 创建挂载点目录，然后用 `sudo mount /dev/sdb1 /mnt/usb` 挂载设备。USB 驱动器的内容现在可以在 `/mnt/usb` 访问。

```bash
sudo mkdir -p /mnt/usb
sudo mount /dev/sdb1 /mnt/usb
```

当你使用完驱动器后，始终使用 `sudo umount /mnt/usb` 正确卸载它，然后再物理断开连接以防止数据损坏。

```bash
sudo umount /mnt/usb
```

### USB 摄像头

USB 摄像头在 `/dev/` 目录中注册为视频设备。运行 `ls /dev/video*` 列出所有可用的视频设备，包括物理摄像头和虚拟视频设备。输出通常会显示 `/dev/video0`、`/dev/video1` 等。

```bash
ls /dev/video*
```

要在 UNO Q 上测试和使用 USB 摄像头，我们可以安装和使用 **Cheese**，一个具有图形界面的用户友好摄像头应用程序。用 `sudo apt install cheese` 安装并用 `cheese` 命令启动它。

```bash
sudo apt install cheese
cheese
```

这会打开一个窗口显示你的摄像头画面，带有拍照或录制视频的选项，使测试摄像头功能变得容易，无需编写代码。Cheese 自动检测并配置你的摄像头，在后台处理所有技术细节。

对于开发项目中的摄像头访问，Python 库如 OpenCV（`python3-opencv`）提供摄像头控制和图像处理能力，与 Arduino App Lab Python 脚本很好地集成。

```bash
sudo apt install python3-opencv
```

### 串口设备

USB 串口设备，包括 Arduino 开发板和 USB 转串口适配器，在系统中显示为 `/dev/` 目录中的特殊设备文件，通常命名为 `/dev/ttyUSB*`（用于通用 USB 串口设备）或 `/dev/ttyACM*`（用于实现 USB 通信设备类的设备）。你可以通过运行 `ls /dev/tty*` 列出所有串口，虽然这会显示许多条目，包括虚拟终端。

要与串口设备交互，`screen` 实用程序提供简单的终端接口。先用 `sudo apt install screen` 安装，然后用 `screen /dev/ttyUSB0 115200` 连接到你的串口设备，其中 115200 是波特率。

这会打开连接到串口的终端会话。要退出 screen，按 **CTRL+A**，然后按 **K**，最后按 **Y** 确认。

```bash
sudo apt install screen
screen /dev/ttyUSB0 115200
```

串口设备通常默认配置有限制性权限。运行 `ls -l /dev/ttyUSB0` 可能显示只有 `dialout` 组中的用户才有访问权限。

```bash
ls -l /dev/ttyUSB0
```

要授予你的用户帐户串口访问权限，用 `sudo usermod -a -G dialout arduino` 将自己添加到 dialout 组。此更改在你注销并重新登录后生效。这一次设置后，你就可以在没有 sudo 权限的情况下访问串口设备。

```bash
sudo usermod -a -G dialout arduino
```

## Arduino App CLI

Arduino UNO Q 预装了 Arduino App CLI（`arduino-app-cli`），这是一个命令行工具，用于直接从终端管理和控制 Arduino App Lab 应用程序。

此工具允许你构建、启动、停止和管理 Apps，而无需使用图形 Arduino App Lab 界面，非常适合自动化、通过 SSH 远程管理或无头操作。

### 管理 Apps

你的 Arduino Apps 存储在 `~/ArduinoApps/` 目录中。导航到那里查看所有可用项目：

```bash
cd ~/ArduinoApps/
ls
```

每个 App 目录包含你的 sketch 代码、Python 脚本和 `app.yml` 配置文件。

当 App 运行时，持久数据保存在 App 目录内的 `data/` 文件夹中，而支持文件（如 Python 虚拟环境）存储在 `.cache/` 文件夹中。

要列出开发板上所有可用的 Apps：

```bash
arduino-app-cli app list
```

要从命令行启动或停止 Apps，使用以下命令格式：

```bash
arduino-app-cli app start user:my-app
```

例如，要启动示例 App：

```bash
arduino-app-cli app start examples:blink
```

要停止正在运行的 App：

```bash
arduino-app-cli app stop user:my-app
```

例如，要停止正在运行的 Blink LED App：

```bash
arduino-app-cli app stop examples:blink
```

### 系统和网络配置

Arduino App CLI 还提供系统管理命令。一个有用的功能是启用或禁用网络模式，这允许 Arduino App Lab 桌面应用程序远程连接到你的开发板。

```bash
arduino-app-cli system network-mode <enable/disable>
```

然后，你可以使用以下命令检查网络模式状态：

```bash
arduino-app-cli system network-mode status
```

网络模式在首次设置期间自动启用，但这些命令让你可以在需要时手动控制它。

### 使用 Apps

对于构建、启动和停止 Apps，**Arduino App Lab**（桌面应用程序或在 SBC 模式下开发板上运行）是推荐的界面。

App Lab 自动处理 Linux 和微控制器组件的编译、部署和执行。

如果你需要手动查看或编辑 App 文件，导航到 App 目录并使用标准 Linux 命令：

```bash
cd ~/ArduinoApps/MyApp
cat app.yaml          # 查看 App 配置
cat sketch/sketch.ino # 查看 Arduino sketch 内容
cat main.py           # 查看 Python 脚本内容
```

> **注意**: 有关全面的 Arduino App CLI 文档，包括创建 Apps、监控日志、管理 Bricks 和系统更新，请参阅专门的 [Arduino App CLI 教程](https://docs.arduino.cc/software/app-lab/tutorials/cli/)。

### Arduino CLI

标准的 Arduino CLI 工具（`arduino-cli`）在 UNO Q 上也可用，供需要在 Arduino App Lab 生态系统之外直接访问 Arduino 核心管理、库安装和 sketch 编译的用户使用。

此工具独立于 Arduino App Lab，遵循标准 Arduino CLI 工作流程。

```bash

```

对于 UNO Q 开发，**Arduino App Lab 和 `arduino-app-cli` 是推荐的工具**，因为它们专门为开发板的架构设计，自动处理 MPU 和 MCU 之间的通信。

标准的 `arduino-cli` 对于需要手动管理核心和库或使用连接到 UNO Q 的其他 Arduino 开发板的用户很有用。

## 系统监控

### 检查系统资源

了解系统的资源使用情况有助于识别性能瓶颈和排除问题。

`top` 命令提供运行进程的交互式实时视图，显示每个进程的 CPU 和内存使用情况。

按 `q` 退出 top 界面。这在追踪哪个程序消耗过多资源时很有用。

```bash
top
```

磁盘空间管理至关重要，特别是在 UNO Q 的 eMMC 存储上。`df -h` 命令以人类可读格式（MB、GB）显示所有已挂载文件系统的磁盘使用情况，显示总大小、已用空间、可用空间和挂载点。

它帮助你监控存储消耗并确定何时空间不足。

```bash
df -h
```

要查看各个目录占用多少空间，在任何目录中使用 `du -sh *`。

这会汇总当前位置每个项目的大小，使识别可能适合清理的大型目录变得容易。

```bash
du -sh *
```

`free -h` 命令显示内存使用统计，以人类可读格式显示总 RAM、已用内存、可用内存和交换空间。`-h` 标志以人类可读格式而不是原始字节计数显示大小。

这帮助你确定应用程序是否消耗过多内存或是否有足够资源运行新程序。

```bash
free -h
```

有几个命令提供有关 UNO Q 硬件和软件配置的详细信息。`uname -a` 命令显示全面的 Linux 内核信息，包括版本、构建日期和架构。

uname -a

要专门查看你正在运行的 Debian 版本，用 `cat /etc/os-release` 检查 `/etc/os-release`，它列出操作系统名称、版本和其他发行详情。

```bash
cat /etc/os-release
```

有关详细的 CPU 信息，包括型号、核心数、时钟频率和支持的功能，使用 `lscpu`。

lscpu

## 网络管理

### 检查网络状态

`ip addr` 命令显示所有网络接口及其 IP 地址、MAC 地址和当前状态。

这显示你的物理接口，如 `wlan0`（Wi-Fi®）和 `eth0`（以太网），以及虚拟接口。输出包括 IPv4 和 IPv6 地址，帮助你确定开发板如何连接到网络。

```bash
ip addr
```

要查看哪些接口当前处于活动或非活动状态，使用 `ip link show`。这显示每个接口的链路状态，指示它是 UP（活动）还是 DOWN（非活动），以及其他属性如 MAC 地址和支持的功能。

```bash
ip link show
```

### Wi-Fi® 管理

NetworkManager 的命令行界面 `nmcli` 提供全面的 Wi-Fi® 管理能力。运行 `nmcli device status` 显示所有网络设备及其当前连接状态。

```bash
nmcli device status
```

要扫描范围内可用的 Wi-Fi® 网络，使用 `nmcli device wifi list`，它显示所有可见网络的 SSID、信号强度、安全类型和信道。

```bash
# 列出可用的 Wi-Fi® 网络
nmcli device wifi list
```

可以使用 `sudo nmcli device wifi connect "SSID" password "password"` 连接到 Wi-Fi® 网络，将 SSID 和密码替换为你的网络凭据。

```bash
nmcli device wifi connect "SSID" password "password"
```

连接会自动保存，允许你的开发板在重启后重新连接。要从当前网络断开，使用 `sudo nmcli device disconnect wlan0`。

你可以用 `nmcli connection show` 查看所有保存的网络连接，它列出连接名称、类型和关联设备。

```bash
nmcli device disconnect wlan0
nmcli connection show
```

### 网络诊断

排除网络问题时，几个诊断工具必不可少。

`ping google.com` 命令通过发送数据包到远程服务器并测量响应时间来测试基本互联网连接。按 **CTRL+C** 停止连续的 ping 输出。

```bash
ping google.com
```

`traceroute google.com` 命令显示数据包到达目的地所经过的网络路径，帮助识别连接问题发生的位置。

```bash
traceroute google.com
```

对于 DNS 故障排除，`nslookup arduino.cc` 查询 DNS 服务器以验证域名正确解析为 IP 地址。

```bash
nslookup arduino.cc
```

#### 通过 ADB 排除网络问题

如果你配置的网络不能正常工作，你可能发现自己无法通过 Arduino App Lab 或网络模式访问开发板。

在这种情况下，你可以使用 ADB 访问 shell 并删除有问题的网络连接，而无需重新刷写开发板或在 SBC 模式下使用键盘和鼠标。

首先，通过 USB-C® 线连接 UNO Q，然后通过 ADB 访问 shell。在 macOS 上，ADB 工具位于你的 Arduino15 目录中：

```bash
# macOS - 通过 ADB 访问 shell
~/Library/Arduino15/packages/arduino/tools/adb/32.0.0/adb shell
```

进入 shell 后，列出保存的连接：

```bash
nmcli connection show
```

使用以下命令删除有问题的连接：

```bash
nmcli connection delete <ConnectionName>
```

在 Linux 上，路径通常是：

```bash
# Linux - 通过 ADB 访问 shell
~/.arduino15/packages/arduino/tools/adb/32.0.0/adb shell
```

在 Windows 上，路径使用用户的 `AppData` 文件夹：

```bash
# Windows - 通过 ADB 访问 shell（在命令提示符或 PowerShell 中）
%LOCALAPPDATA%\Arduino15\packages\arduino\tools\adb\32.0.0\adb.exe shell
```

删除错误的连接后，你可以退出 ADB shell，通过 [Arduino App Lab 的首次设置流程](https://docs.arduino.cc/tutorials/uno-q/user-manual/#install-arduino-app-lab) 或使用上述命令连接到可工作的网络来重新配置网络。

> **注意**: 当你被锁定在网络访问之外且没有可用于 SBC 模式的 USB-C® 转接器、键盘或鼠标时，此 ADB 方法很有用。

## 实用技巧和最佳实践

### 命令行效率

终端包含多个有助于提高效率的键盘快捷键。

按 **Tab** 自动完成命令和文件名，节省输入并防止错误。

要停止运行中的命令，按 **CTRL + C**。要清除终端屏幕，使用 **CTRL + L**。

`up` 和 `down` 箭头键让你浏览之前输入的命令，便于重复或修改最近的命令。反向搜索功能（按 **CTRL + R** 激活）允许你通过输入几个字符搜索整个命令历史。

### 获取帮助

Debian 通过手册系统为几乎每个命令包含文档。

运行 `man <command-name>` 打开任何命令的详细手册页，包括所有选项的解释、使用示例和相关命令。

```bash
man <command-name>
man ls
```

要更快参考，大多数命令支持 `--help` 标志，如 `ls --help`，它在终端直接显示选项摘要。

```bash
ls --help
```

### 系统日志和故障排除

系统日志对于诊断问题很重要。

`journalctl -xe` 命令显示最近的系统日志条目及其解释性文本，帮助你了解出问题时发生了什么。

```bash
journalctl -xe
```

对于实时日志监控，`journalctl -f` 在日志写入时跟踪它们，类似于观看系统事件的实时源。

```bash
journalctl -f
```

`dmesg | less` 命令显示内核启动消息和硬件检测日志，这对排除硬件问题或驱动程序问题很有帮助。

```bash
dmesg | less
```

### 安全系统管理

遵循最佳实践有助于防止系统损坏。**理解何时使用 `sudo`** 对安全系统管理很重要。你应该只在需要系统级操作的提升权限时使用 `sudo`，如用 `apt` 安装软件包、编辑 `/etc/` 中的系统配置文件或访问受保护目录。

**对于在主目录创建文件、编辑自己的文档或运行程序等日常任务，不需要也不应该使用 `sudo`。**

不必要地使用 `sudo` 实际会造成问题。当你用 `sudo` 创建或修改文件时，它们由 **root** 而非你的用户帐户拥有，这可能导致以后的权限问题。

例如，如果你在主目录运行 `sudo nano myfile.txt`，文件将由 root 拥有。你以后需要 `sudo` 才能再次编辑它。这就是为什么你应该保留 `sudo` 用于真正需要系统管理员权限的操作。

在进行重大系统更改之前，始终将重要文件备份到 **外部存储设备**，如你工作过的 Apps。这个简单的预防措施可以在出现问题时节省数小时的工作。

如果可用，使用命令选项在空运行模式下运行并在实施前预览更改，允许你在不实际修改任何内容的情况下验证操作。

定期运行 `sudo apt update && sudo apt upgrade` 保持系统和软件包最新，获取最新的安全补丁、兼容性元素和错误修复。

要特别谨慎使用 `rm -rf` 等命令，因为它们永久删除文件而不确认，无法撤销。在执行破坏性命令之前始终仔细检查路径和文件名。同样，除非必要，避免在文件或目录上使用 `chmod 777`，因为这授予所有人完全访问权限并造成安全漏洞。

当对命令效果不确定时，用 `man <command-name>` 查阅手册或在继续前搜索示例。花这些额外时间验证你的操作可以防止因过早错误带来的沮丧和数据丢失。

### 安全关闭 UNO Q

与传统计算机不同，UNO Q 具有自动重启功能。当你运行标准的 Linux 关机命令如：

```bash
sudo shutdown now
# 或
sudo poweroff
```

开发板会干净地关闭 Debian 系统，但随后会自动重启。此行为内置在开发板的电源管理系统中。

要安全关闭 UNO Q 以进行长期存储或携带开发板，你需要使用正确的命令安全关闭开发板并保持其关机状态：

```bash
sudo halt
```

#### 推荐的关机方法

`halt` 命令是安全关闭 UNO Q 的推荐方法。它停止所有系统进程，将 Linux 置于安全状态，并保持开发板关机，防止自动重启。

要正确关闭 UNO Q，运行：

```bash
sudo halt
```

当系统完全停止时，开发板上的绿色电源 LED 会熄灭。开发板将保持关机状态，不会自动重启。

这使得 `sudo halt` 非常适合长期存储、携带开发板或任何你希望开发板保持关闭直到手动重新上电的情况。

#### 替代关机方法

以下关机方法会干净地关闭 Debian 系统，但会触发开发板的自动重启功能：

```bash
sudo shutdown now
sudo poweroff
```

使用图形界面方法（SBC 模式）：
- 点击桌面右上角的 **"arduino"**
- 从出现的下拉菜单中选择 **"关机"** 选项

使用任何这些方法时，开发板会执行干净的关机，然后在几秒钟内自动重启。

如果你需要在使用这些方法时保持开发板关机，必须在绿色 LED 熄灭后、重启序列开始之前立即断开电源。

使用这些自动重启方法时，时机很重要。LED 熄灭后你只有短暂的时间窗口来安全断开电源。如果你等待太久，系统将开始其重启序列。

如果是通过电源断开：
- 对于通过 USB-C® 供电的开发板，在 LED 熄灭后拔掉 USB-C® 线缆
- 使用 VIN 引脚供电（7-24 VDC 输入）时，此时断开外部电源
- 如果你的开发板从 5 V 引脚接收电源，在 LED 指示灯熄灭时断开 5 V 电源

在带有 USB-C® 转接器的 SBC 模式下，在 LED 熄灭后断开转接器的电源。

#### 紧急关机和最佳实践

虽然正确的关机程序可以防止文件系统损坏和数据丢失，但 UNO Q 的 eMMC 存储包含保护措施，可以在意外断电时最小化数据丢失。

如果你必须在紧急情况下立即断开电源，系统很可能在下次启动时恢复。但是，任何未保存的工作可能会丢失。

对于长期存储或携带开发板，使用 `sudo halt` 命令正确关机：

```bash
sudo halt
```

对于连续运行或自动化系统，开发板无限期运行，不需要手动关机程序。

#### 理解关机命令

理解不同关机命令的行为有助于你为情况选择正确的方法。

以下命令按预期干净地重启系统：

```bash
sudo reboot
```

以下命令安全关闭开发板并保持其关机，这是这种情况的推荐方法：

```bash
sudo halt
```

以下命令执行干净的系统关机，但开发板的自动重启功能会在关机完成后不久重新启动系统：

```bash
sudo shutdown now
sudo poweroff
```

使用 `sudo halt` 保持开发板关机。仅在你特别希望开发板重启，或准备好快速断开电源以防止自动重启时使用其他关机方法，具体取决于某些应用要求。
