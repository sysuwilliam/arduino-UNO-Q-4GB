# 官方文档补充说明

在阅读这个示例时，有几个点值得在官方 README 基础上额外补充说明。

## 1. 这不是传统意义上的 Arduino 草图示例

很多初学者看到 Arduino 示例，会默认寻找：

- `sketch.ino`

但这个项目没有 `sketch.ino`。  
原因是它的核心功能并不依赖 MCU GPIO，而是依赖：

- Python 后端
- Web 前端
- AI Brick
- 手机远程摄像头

所以它更像是一个 App Lab 全栈示例，而不是经典 Arduino 硬件控制例程。

## 2. 二维码不仅仅是“展示用”

二维码并不是一个装饰性的前端功能，它实际上承担了“把手机与 UNO Q 配对”的关键任务。  
二维码中包含了：

- 密钥
- 协议
- IP
- 端口

因此二维码是这个项目连接流程的核心之一。

## 3. 手机和开发板必须在同一网络

这是这个示例最容易被忽略的现实前提。  
如果两者不在同一局域网，即使代码本身没问题，也无法完成正常配对和视频流传输。

## 4. 这个项目是一个事件驱动系统

它的运行不是“顺序脚本一路跑到底”，而是大量依赖事件：

- 浏览器连接事件
- 手机连接事件
- 视频流状态变化事件
- 检测结果事件
- 阈值调节事件

理解这一点之后，阅读 `main.py` 和 `app.js` 会更容易。

## 5. 这是一个很适合继续扩展的示例

这个示例可以比较容易地扩展为：

- 手机视频流 + 自定义模型识别
- 手机视频流 + 自定义反馈 UI
- 手机视频流 + 识别后联动 GPIO 或 LED 矩阵
- 手机视频流 + 数据存储 / 报警系统

也就是说，它很适合作为后续 AI 项目的基础模板。
