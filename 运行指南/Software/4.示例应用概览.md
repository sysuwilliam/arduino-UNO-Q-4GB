Arduino App Lab 平台开发的各种应用概览，展示从环境监测到机器学习的独特功能。

**作者**：Karl Söderby, Christopher Méndez, Ernesto Voltaggio, Pedro Lima

## 概述

本文提供了 Arduino App Lab 各种应用的全面概览。每个应用都展示了独特的功能，从环境监测到机器学习，充分利用 Arduino 生态系统的能力。以下是每个示例的摘要，详细说明了 App 的用途、功能和使用的组件。

## 无需额外硬件的示例

### 空气质量监测 App

![Air Quality Monitoring App](https://docs.arduino.cc/static/7e543ff68a2962354817121667ff4937/a6d36/air-quality-monitoring-app-hero.png)

**空气质量监测 App** 在 8x13 LED 矩阵上使用 AQICN 服务显示实时空气质量数据。它通过动画表情图案直观地表示空气质量水平，范围从"良好"到"危险"。

[**示例源代码**](https://github.com/arduino/app-bricks-examples/tree/main/examples/air-quality-monitoring)

#### 工作原理

- **数据获取**：App 从 AQICN API 获取指定城市的空气质量数据。
- **可视化**：数值 AQI 值转换为在 LED 矩阵上显示的视觉图案。
- **通信**：Python® 脚本处理 API 通信，而 Arduino sketch 管理 LED 显示。

#### 使用的 Bricks

- **无**：Python® 和 Arduino 之间的直接通信通过 Router Bridge 实现。

### 睡前故事生成器

![Bedtime Story Teller Example](https://docs.arduino.cc/static/5ace96b2d07973b3a186142675b5a4dc/a6d36/bedtime-storyteller.png)

**睡前故事生成器**示例演示如何使用 Arduino UNO Q 构建生成式 AI 应用。它使用大语言模型（LLM）根据用户选择的参数（如年龄、主题和角色）创建个性化睡前故事，并将结果实时流式传输到 Web 界面。

**注意：** 此示例需要 LLM 提供商（Google Gemini、OpenAI GPT 或 Anthropic Claude）的有效 API 密钥。

[**示例源代码**](https://github.com/arduino/app-bricks-examples/tree/main/examples/bedtime-story-teller)

#### 工作原理

- **用户输入收集**：Web 界面收集故事参数（年龄、主题、基调）或触发随机生成。
- **AI 推理**：后端构建提示并通过 `cloud_llm` Brick 发送到云端提供商。
- **流处理**：故事按文本令牌流式返回浏览器，提供即时反馈。

#### 使用的 Bricks

- **cloud_llm**：用于与云端大语言模型交互的 Brick。
- **web_ui**：用于创建参数输入和故事显示的 Web 界面的 Brick。

[**示例源代码**](https://github.com/arduino/app-bricks-examples/tree/main/examples/theremin)

#### 额外硬件

- USB 音频设备（扬声器或无线接收器）
- 带外部电源的 USB-C® 集线器

#### 工作原理

- **用户交互**：前端捕获鼠标/触摸坐标并通过 WebSocket 发送到后端。
- **合成**：`wave_generator` Brick 应用包络平滑并将生成的正弦波流式传输到 USB 音频设备。

#### 使用的 Bricks

- **web_ui**：提供 Web 界面和用于实时控制的 WebSocket 通道的 Brick。
- **wave_generator**：处理音频合成、包络控制和流式传输到 USB 音频设备的 Brick。

### 振动异常检测

**振动异常检测**示例实时监控机械（例如风扇）的物理状态。它可视化原始加速度计数据，并允许用户动态调整异常检测的灵敏度。

#### 额外硬件

#### 工作原理

- **采集**：系统读取传感器数据并将其转换为检测算法。
- **检测**：`vibration_anomaly_detection` Brick 分析模式，如果振动超过用户定义的阈值则触发事件。

#### 使用的 Bricks

- **web_ui**：用于创建显示仪表板的 Web 界面的 Brick。
- **vibration_anomaly_detection**：处理加速度计数据以检测不规则振动模式的 Brick。

### 移动端视频通用目标检测

![Mobile Video Generic Object Detection](https://docs.arduino.cc/static/2685a6ecbc249f9f37a4e357b2840117/a6d36/mobile-video-generic-object-detection.png)

**移动端视频通用目标检测**示例支持智能手机摄像头画面上的实时目标检测。使用 Arduino IoT Remote 应用将视频从手机流式传输到 Arduino UNO Q，开发板通过 AI 模型处理视频，并在 Web 界面中显示检测到的对象周围的边界框。

[**示例源代码**](https://github.com/arduino/app-bricks-examples/tree/main/examples/mobile-video-generic-object-detection)

#### 工作原理

- **移动端流传输**：Arduino IoT Remote 应用使用安全配对二维码通过同一网络将智能手机摄像头的实时视频流式传输到 Arduino UNO Q 开发板。
- **AI 目标检测**：开发板通过目标检测 AI 模型处理视频帧，该模型可以识别 80 多个对象类别，包括猫、狗、杯子和手机。
- **Web 界面**：显示带有检测对象周围边界框的实时视频画面、置信度分数，并允许用户通过交互式滑块调整检测阈值。
- **实时结果**：显示最近五次检测及其对象类别、置信度分数和时间戳。

#### 使用的 Bricks

- **web_ui**：用于创建显示结果、控件和配对二维码的交互式 Web 界面的 Brick。
- **video_objectdetection**：用于通过目标检测 AI 模型处理实时视频帧的 Brick。

### 彩色 LED 控制

![Color Your LEDs](https://docs.arduino.cc/static/8d211d40cea8299d5bf1beba67472e8c/a6d36/color-your-leds.png)

**彩色 LED 控制**示例允许通过基于 Web 的界面交互式控制 Arduino UNO Q 上的四个内置 RGB LED。它演示了管理混合架构，其中不同的 LED 连接到开发板上的不同处理器。

[**示例源代码**](https://github.com/arduino/app-bricks-examples/tree/main/examples/color-your-leds)

#### 工作原理

- **混合架构**：LED 1-2 连接到 Qualcomm QRB2210 MPU，通过直接 API 调用控制；LED 3-4 连接到 STM32U585 MCU，通过 RouterBridge 通信控制。
- **Web 仪表板**：Web 界面为每个 LED 提供开/关状态切换开关和颜色选择器。
- **颜色控制**：LED 1、2 和 4 提供基本颜色选项，而 LED 3 由于 PWM 功能提供完整的调色板，可实现精确的 RGB 混合。
- **实时更新**：当用户从浏览器界面选择颜色时，Socket.IO 连接启用即时 LED 控制响应。

#### 使用的 Bricks

- **web_ui**：用于创建可通过浏览器访问的交互式颜色控制仪表板的 Brick。
