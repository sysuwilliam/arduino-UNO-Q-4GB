# Arduino App Lab 与 IoT Remote 手机 App 集成

本教程教您如何利用 Arduino IoT Remote 应用程序在 Arduino App Lab 应用程序中使用手机传感器。

**作者**：Christopher Méndez  
**最后修订**：2026年3月3日

---

## 概述

在本教程中，您将学习如何使用 IoT Remote 应用程序在 Arduino App Lab 中解锁智能手机的潜力。我们将探索一个强大的新功能，将您的手机变成无线输入设备。具体来说，您将学习如何将视频从手机直接流式传输到 Arduino App Lab 以为 Object Detection Brick 提供支持，让您无需 USB 网络摄像头即可运行 AI 视觉项目。

![thumbnail](https://docs.arduino.cc/static/e1f248d770f28a342675753365be2262/a6d36/thumbnail.png)

**注意：** 您的智能手机将用作远程摄像头输入。Arduino UNO Q 和您的智能手机必须连接到同一网络。

---

## 目标

- 理解 IoT Remote 应用程序和 Arduino App Lab 之间的集成。
- 配置您的智能手机作为项目的无线摄像头输入。
- 使用 Object Detection Brick 测试示例应用程序，以远程视频流作为输入。
- 运行项目以使用手机实时检测和分类对象。

## 工作原理

此集成通过安全配对过程将您的智能手机变成 Arduino App Lab 的无线摄像头：

### 配对过程

1. **二维码生成**：当您运行应用程序时，Arduino UNO Q 生成唯一的一次性密码（OTP）并创建包含连接详细信息（IP 地址、端口和密钥）的二维码。

2. **安全握手**：您使用手机摄像头应用程序扫描二维码。IoT Remote 应用程序自动打开并接收密钥，通过安全的 WebSocket 连接向您的开发板进行身份验证。

3. **视频流传输**：一旦通过身份验证，您的手机开始通过 HTTP 在端口 `4912` 上流式传输视频。开发板接收这些帧并使其可用于您的应用程序。

### 架构概览

系统使用两个通信通道：

- **WebSocket（控制通道）**：处理初始配对并发送控制消息。每个会话使用唯一的密钥以确保安全。
- **HTTP（数据通道）**：在端口 `4912` 上传输实际视频流，提供低延迟的持续数据传输。

***两个设备直接通过您的本地网络通信。没有视频数据发送到云端，确保隐私和低延迟。***

## 移动摄像头流传输

要测试此功能，我们将利用 Arduino App Lab 内的 **Detect Objects on Smartphone Camera** 示例。它将让我们轻松学习此功能的工作原理并试用它，而无需从头开发 App。

![Detect Objects on Smartphone Camera](https://docs.arduino.cc/static/080df1f2213fb31014decadde911638d/a6d36/mobile-example.png)

***要将手机摄像头流传输到 UNO Q，两者必须在同一网络上。***

***注意：视频流通过 HTTP 在端口 4912 上传输。如果您在公司或受限网络上，请确保此端口可访问。初始配对使用动态分配端口上的单独 WebSocket 连接。***

### Arduino App Lab 设置

1. 确保您的 Arduino UNO Q 已通电并连接到网络。

2. 在计算机上打开 Arduino App Lab。

3. 在 Arduino App Lab 中运行 **Detect Objects on Smartphone Camera** 示例。

4. 应用程序应在 Web 浏览器中自动打开。您可以通过 `<board-name>.local:7000` 手动打开它。

5. Web UI 将显示一个 **QR Code**。

![QR Code in the browser](https://docs.arduino.cc/static/5afb29478f4438d337fd763e227f53db/a6d36/qr-code-full.png)

### Arduino IoT Remote 设置

1. 确保在智能手机上安装了 [**Arduino IoT Remote**](https://cloud.arduino.cc/iot-remote-app/) 应用程序。

![App available in your favorite OS](https://docs.arduino.cc/static/4a092a5eb37d253b3a7a63f7832a8f64/a6d36/install-app.png)

2. 使用手机摄像头应用程序扫描二维码。

![QR code scan](https://docs.arduino.cc/static/ffd04c96693c98eb0bebf3f26b403559/a6d36/qr-code.png)

3. Arduino IoT Remote 应用程序将在您的手机上自动打开。如果尚未登录，请使用 Arduino 账户登录。

4. 连接后，摄像头画面将直接在手机上可见，视频流将传输到 Web UI。

5. 将手机对准对象，观看应用程序检测和识别它们。

![Phone's camera stream](https://docs.arduino.cc/static/1e340e57d465dd06af79dac0c997246e/a6d36/camera-stream.png)

### 无二维码配置（可选）

您也可以从 IoT Remote 应用程序配置手机，而无需扫描二维码：

1. 转到 Devices，点击加号图标设置新设备，选择 **Stream phone camera to UNO Q**。

![IoT Remote setup](https://docs.arduino.cc/static/39dc21f7803419a66f1f091d31c4fb61/a6d36/iot-remote.png)

2. 从列表中选择您的 Arduino UNO Q。（它必须连接到同一网络）

![Select your UNO Q](https://docs.arduino.cc/static/cf70e69457e0e9c062e875fef90e2146/a6d36/search-uno-q.png)

3. 在 Web UI 上的二维码下方输入 6 位代码。

![Start streaming](https://docs.arduino.cc/static/17769a0563dfdada2c02d88bc1a1f5ad/a6d36/start-streaming.png)

4. 流传输将立即开始。

## 移动集成功能

要将智能手机摄像头集成添加到您自己的自定义 Arduino App Lab 应用程序中，您需要在开发板和手机之间实现特定的握手机制。它由管理安全连接的 Python® 后端和生成配对二维码的 JavaScript 前端组成。

### 1. 后端实现（`main.py`）

后端负责创建一个安全的"房间"供手机连接。使用 `WebSocketCamera` 类生成一次性密码（密钥），仅允许预期的手机连接到您的开发板。

**必需的导入：**

```
import secrets
import string
from arduino.app_peripherals.camera import WebSocketCamera
```

**设置逻辑：**

您必须生成随机密钥（OTP）并将其传递给 `WebSocketCamera` 实例。您还需要将这些连接详细信息发送到前端，以便生成二维码。

```
# 1. Generate a random 6-digit secret for security
def generate_secret() -> str:
    characters = string.digits
    return ''.join(secrets.choice(characters) for _ in range(6))

secret = generate_secret()

# 2. Initialize the Camera with the secret
# 'encrypt=True' ensures the handshake is secure
resolution = (480, 640) # Portrait resolution for mobile devices
camera = WebSocketCamera(resolution=resolution, secret=secret, encrypt=True, adjustments=resized(resolution, maintain_ratio=True))

# 3. Send connection details to the Frontend when a client connects
# This passes the IP, Port, and Secret required for the QR Code
ui.on_connect(lambda sid: ui.send_message("welcome", {
    "client_name": camera.name,
    "secret": secret,
    "status": camera.status,
    "protocol": camera.protocol,
    "ip": camera.ip,
    "port": camera.port
}))
```

### 2. 前端实现（`app.js`）

前端充当桥梁。它从后端接收连接详细信息并将其编码为二维码内的特定 URL 格式。当 Arduino IoT Remote 应用程序扫描此码时，它确切知道将视频流发送到哪里。

**先决条件：** 确保您的 HTML 中包含二维码库（例如 `qrcode.min.js`）。

**处理握手：**

在您的 `app.js` 中，监听来自后端的"welcome"消息。使用接收到的数据生成配对 URL。

```
// Listen for connection details from main.py
socket.on('welcome', async (message) => {
    // If the camera is not yet connected, generate the QR code
    if (message.status !== "connected") {
        generateQRCode(message.secret, message.protocol, message.ip, message.port);
    }
});

function generateQRCode(secret, protocol, ip, port) {
    const qrCodeContainer = document.getElementById('qrCodeContainer');
    
    // The specific URL format required by the IoT Remote App
    const connectionUrl = `https://cloud.arduino.cc/installmobileapp?otp=${secret}&protocol=${protocol}&ip=${ip}&port=${port}`;

    new QRCode(qrCodeContainer, {
        text: connectionUrl,
        width: 128,
        height: 128
    });
}
```

**显示流：**

一旦手机连接，视频不是通过 WebSocket 发送，而是通过 HTTP 在特定端口（默认通常为 `4912`）上提供。您应该使用 `<iframe>` 来显示它。

```
// Logic to display video when status becomes 'streaming'
const streamUrl = `http://${window.location.hostname}:4912/embed`;
document.getElementById('videoIframe').src = streamUrl;
```

## 故障排除提示

- **二维码无法扫描**：屏幕亮度低或光照条件差可能是原因。尝试增加计算机屏幕亮度并确保二维码光照良好。将手机稳定地保持在距屏幕舒适距离（10-15 厘米）处。

- **"Connection failed"错误**：手机和 Arduino UNO Q 可能在不同网络上。请验证两个设备都连接到同一 Wi-Fi® 网络。检查手机的 Wi-Fi 设置并确保您没有使用移动数据。

- **视频流无法加载**：端口 4912 可能被防火墙或网络限制阻止。检查计算机的防火墙设置并确保端口 4912 对传入连接开放。如果您在公司网络上，可能需要向 IT 管理员请求访问权限。

- **视频卡顿或停顿**：网络拥塞或带宽不足可能影响流质量。尝试将后端代码中的分辨率降低到 `(320, 480)` 以获得更好的性能。确保流传输期间没有其他设备大量使用网络。

- **"Invalid secret"消息**：二维码可能已过期（通常在 5 分钟后或一次成功连接后）。刷新网页以生成带有新密钥的新二维码，然后重新扫描。

- **流突然停止或冻结**：您的手机可能已进入睡眠状态或失去网络连接。通过调整 IoT Remote 应用程序中的睡眠设置保持手机唤醒，或偶尔点击屏幕。验证手机上的 Wi-Fi® 未断开连接。

在本教程中，您学习了如何使用 Arduino IoT Remote 应用程序将智能手机变成 Arduino App Lab 的无线输入设备。您成功配置了网络环境，通过二维码将手机与 UNO Q 配对，并流式传输实时视频以驱动 AI 对象检测模型。

此集成消除了对外部 USB 网络摄像头的需求，让您可以更自由地原型设计计算机视觉应用程序。通过理解 Python® 后端和 JavaScript 前端之间的握手机制，您现在拥有构建利用移动设备中已有强大传感器的自定义应用程序的基础。
