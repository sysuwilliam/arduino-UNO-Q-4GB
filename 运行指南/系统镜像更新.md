# 刷写新镜像到 UNO Q

学习如何使用 Arduino Flasher CLI 将新镜像（Linux OS）刷写到 UNO Q 开发板。

**作者**：Karl Söderby  
**最后修订**：2026年5月2日

[Arduino® UNO Q](https://store.arduino.cc/products/uno-q) 运行 Linux 操作系统（OS），该系统已预装在开发板上。

通常情况下，无需在开发板上重新安装 OS（也称为刷写镜像），因为 OS 会定期自动接收更新。

但是，如果我们想要对开发板进行硬重置并执行全新安装，这是可行的。此过程需要在您的计算机上安装一些额外的工具。

**警告！** 请注意，本教程中的说明将清除开发板上的所有数据，保存在开发板上的文件和配置将被删除。

---

## 要求

### 硬件要求

- [Arduino® UNO Q](https://store.arduino.cc/products/uno-q)
- [USB-C® 数据线](https://store.arduino.cc/products/usb-cable2in1-type-c)
- 母对母跳线

### 软件要求

- [Arduino Flasher CLI](https://www.arduino.cc/en/software/#flasher-tool)
- 至少 10 GB 的可用磁盘空间

要重新刷写开发板，我们将使用 **Arduino Flasher CLI** 工具：

- 下载 [Arduino Flasher CLI](https://www.arduino.cc/en/software/#flasher-tool)

***请注意，此工具将下载大小超过 1 GB 的镜像。建议使用稳定的网络连接和至少 10 GB 的可用磁盘空间。***

## Linux 用户：需要 USB 权限

如果您使用的是 Linux，建议在尝试刷写开发板之前配置 USB 设备权限。刷写过程通过 USB 上的 ADB（Android Debug Bridge）与开发板通信，这需要对开发板连接到计算机时创建的 USB 设备文件具有写入权限。

如果没有适当的权限，刷写过程将失败并显示以下错误消息：

```
adb: insufficient permissions for device
```

![udev Rules (1)](https://docs.arduino.cc/static/6f7a1796ae11d9a5993ff1d6f2521d90/a6d36/udev_rules_1.png)

这是因为 UNO Q 在刷写过程中以一种特殊模式工作，称为紧急下载模式（Emergency Download Mode），即 EDL。此模式使用与常规操作不同的 USB 配置。

在常规操作中，开发板以 USB VID `2341` 和 PID `0078` 标识自己。当进入 EDL 模式进行刷写或恢复操作时，它会切换到 USB VID `05c6` 和 PID `9008`。您的 Linux 系统需要识别并授予这两种模式的权限，这就是我们需要为两种配置安装 udev 规则的原因。

### 安装所需的 Udev 规则

为您的用户账户配置必要权限的正确方法是通过 `udev rules`。以下 udev 规则将授予您的用户账户在两种操作模式下访问 UNO Q 的权限：

```
# Operating mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="2341", ATTRS{idProduct}=="0078", MODE="0660", TAG+="uaccess"
# EDL mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="05c6", ATTRS{idProduct}=="9008", MODE="0660", TAG+="uaccess"
```

运行以下命令安装必要的 udev 规则：

```
echo \
'# Operating mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="2341", ATTRS{idProduct}=="0078", MODE="0660", TAG+="uaccess"
# EDL mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="05c6", ATTRS{idProduct}=="9008", MODE="0660", TAG+="uaccess"' \
| \
  sudo \
 tee \
      "/etc/udev/rules.d/60-Arduino-UNO-Q.rules" \
&& \
sudo \
 udevadm control \
    --reload-rules \
&& \
sudo \
 udevadm trigger
```

![udev Rules (2)](https://docs.arduino.cc/static/31b990a2b0b3ec1d1bf67e0dfae97d44/a6d36/udev_rules_2.png)

此命令创建一个 udev 规则文件，告诉 Linux 在 UNO Q 连接时自动授予访问权限。`MODE="0660"` 参数设置文件权限，以便所有者和组可以对设备进行读写。`TAG+="uaccess"` 参数专门授予当前登录用户账户的访问权限。

创建规则后，该命令会重新加载所有 udev 规则而无需重启系统，然后将新规则应用于任何已连接的设备。这意味着如果您的 UNO Q 当前已插入，它将立即获得正确的权限。

您可以使用以下命令验证安装：

```
cat /etc/udev/rules.d/60-Arduino-UNO-Q.rules
```

![udev Rules (3)](https://docs.arduino.cc/static/21c2b71b69854468cbfd782098a0642b/a6d36/udev_rules_4.png)

您应该看到两个 udev 规则，一个用于操作模式，一个用于 EDL 模式。

要检查 UNO Q 是否被检测到，请使用以下命令：

```
lsusb | grep -E "2341:0078|05c6:9008"
```

![udev Rules (4)](https://docs.arduino.cc/static/c25ed64c4d33e7c00665e0180fbdc298/a6d36/udev_rules_5.png)

***有关 Linux USB 权限和故障排除的更多信息，请参阅用户手册中的 [Linux Host Setup 部分](https://docs.arduino.cc/tutorials/uno-q/user-manual/#linux-host-setup-required-for-linux-users)。***

安装 udev 规则后，**将 UNO Q 开发板从计算机上拔下，等待几秒钟，然后重新连接**。这有助于应用新权限。重新连接后，您可以继续刷写过程的下一步。

## 替代安装方法

您可以使用 [*Arduino Core Zephyr 仓库*](https://github.com/arduino/ArduinoCore-zephyr) 中的官方 `post-install` 脚本自动配置这些权限。此脚本执行与上述手动方法相同的 udev 规则安装。

要下载并运行 `post-install` 脚本，请导航到您的 Downloads 目录并使用 `wget` 从仓库获取脚本：

```
cd ~/Downloads
```

```
wget https://raw.githubusercontent.com/arduino/ArduinoCore-zephyr/main/post_install.sh
```

![udev Rules (5)](https://docs.arduino.cc/static/41b9f49d7b9fa525d5d58b2742a6d135/a6d36/udev_rules_7.png)

```
chmod +x post_install.sh
```

```
sudo ./post_install.sh
```

![udev Rules (6)](https://docs.arduino.cc/static/36a3e8d280f595e27a5870887c9d42af/a6d36/udev_rules_8.png)

如果您的系统上没有 wget，可以使用 `curl` 代替：

```
cd ~/Downloads
```

```
curl -O https://raw.githubusercontent.com/arduino/ArduinoCore-zephyr/main/post_install.sh
```

```
chmod +x post_install.sh
```

```
sudo ./post_install.sh
```

```
sudo ~/.arduino15/packages/arduino/hardware/zephyr/<version>/post_install.sh
```

或者，您可以克隆整个仓库并从那里运行脚本：

```
cd ~/Downloads
```

```
git clone https://github.com/arduino/ArduinoCore-zephyr.git
```

```
cd ArduinoCore-zephyr
```

```
chmod +x post_install.sh
```

```
sudo ./post_install.sh
```

当提示时，输入您的密码。脚本将创建 udev 规则文件、重新加载 udev 规则并将其应用于任何当前连接的设备，就像手动安装方法一样。

运行脚本后，您可以使用与手动安装部分相同的命令验证安装：

```
cat /etc/udev/rules.d/60-arduino-zephyr.rules
```

![udev Rules (7)](https://docs.arduino.cc/static/15ab7e2f9d6bcdb59abd315d694acf3e/a6d36/udev_rules_9.png)

请注意，脚本创建的文件名为 `60-arduino-zephyr.rules` 而不是 `60-Arduino-UNO-Q.rules`，但它具有相同权限规则用于两种操作模式。

使用任一方法安装 udev 规则后，**将 UNO Q 开发板从计算机上拔下，等待几秒钟，然后重新连接**。这允许应用新权限。重新连接后，您可以继续刷写过程的下一步。

## 验证 ADB 访问

在尝试刷写镜像之前，您应该检查 ADB 是否能够无权限错误地检测到您的开发板。运行以下命令：

```
adb devices
```

此命令列出连接到您系统的所有 Android Debug Bridge 设备。您的 UNO Q 应出现在列表中且没有任何错误消息。或者使用以下命令访问 UNO Q：

```
adb shell
```

![udev Rules (5)](https://docs.arduino.cc/static/2367909092d869f4d4ccf0817715652c/a6d36/udev_rules_6.png)

如果在输出中看到 `insufficient permissions`，这表示 udev 规则未正确应用或开发板需要重新连接。

如果权限错误仍然存在，请仔细检查 udev 规则是否安装到正确位置。检查您是否在安装规则后拔下并重新连接了开发板，因为现有连接不会自动应用新权限。确保您已关闭并重新打开安装规则时运行的任何终端或应用程序。

如果您仍然遇到问题，请尝试重启 udev 服务：

```
sudo systemctl restart systemd-udevd
```

重启服务后，拔下开发板，等待几秒钟，然后重新连接。然后再次运行 `adb devices` 或 `adb shell` 验证开发板是否被正确检测。一旦 ADB 无错误显示您的设备，您就可以放心地继续刷写步骤。

## 下载并安装 CLI 工具

1. 为您的操作系统下载 [Arduino Flasher CLI](https://www.arduino.cc/en/software/#flasher-tool)（MacOS / Linux / Windows）

2. 解压下载的文件（您将收到一个名为 `arduino-flasher-cli` 的可执行二进制文件）

在刷写新镜像之前，请检查 `arduino-flasher-cli` 工具是否正常工作。以下是验证工具在各操作系统上已安装并正常工作的说明。

这表示工具正常工作，我们可以继续[准备硬件](#准备硬件)。

## 修复 qcserial 问题（仅限 Linux）

打开终端，通过运行 `lsmod | grep qcse` 检查 `qcserial` 是否存在。

如果存在，它将返回：

```
> lsmod | grep qcse
qcserial 24576 0
usb_wwan 24576 1 qcserial
usbserial 69632 2 qcserial,usb_wwan
```

然后，通过运行 `sudo dmesg` 检查 `qcserial` 是否锁定了串口 `ttyUSB0`：

```
> sudo dmesg
[31633.372270] qcserial ttyUSB0: Qualcomm USB modem converter now disconnected from ttyUSB0
[31633.372308] qcserial 3-3:1.0: device disconnected
```

此问题可以通过在 Linux 系统上禁用（或黑名单）`qcserial` 内核模块来解决。

1. 在 `/etc/modprobe.d/` 目录中创建或编辑名为 `blacklist-modem.conf` 的配置文件：

```
sudo nano /etc/modprobe.d/blacklist-modem.conf
```

2. 在此文件中，我们需要将 `qcserial` 模块加入黑名单：

```
blacklist qcserial
```

3. 保存文件并重启系统以使配置生效。

## 刷写镜像到开发板

在此步骤中，我们将使用 Arduino Flasher CLI 将新镜像上传到开发板。

1. 使用 USB-C® 类型数据线将开发板连接到计算机。

2. 打开终端并导航到解压 Arduino Flasher CLI 的目录（通常是 `cd /Downloads`）。

3. 在终端中运行以下命令：`./arduino-flasher-cli flash latest`。

4. 下载序列将开始（镜像大小 >1 GB）。下载完成后，它将使用新镜像刷写开发板。**请注意：** 这将需要几分钟时间。在此过程中**不要**断开 USB 数据线。

5. 刷写完成且工具报告成功后，移除上图所示两个引脚之间的母对母跳线，然后**重启电源**（拔下并重新插入 USB）以便启动新 OS。

## 故障排除

- 确保 Arduino Flasher CLI 已解压且可访问。

- 检查正确的引脚是否已短接，并且它们是在将开发板连接到计算机**之前**短接的。

- 确保运行刷写命令（`./arduino-flasher-cli flash latest`）时开发板尚未完成启动。

- 验证您是从解压 Arduino Flasher CLI 的正确目录运行命令。

- 工具报告安装成功后，**在移除跳线的情况下**重启开发板电源。
