# Arduino UNO Q 远程访问选项指南

学习如何使用 Tailscale、xrdp 或 RustDesk 从任何地方远程访问你的 UNO Q。

**作者**: Ernesto Voltaggio  
**最后修订**: 2026年4月14日

---

本教程介绍如何从本地网络外部远程访问你的 [Arduino® UNO Q](https://store.arduino.cc/products/uno-q)。我们将介绍三种方法：

1. **Tailscale + SSH**：从任何地方进行安全的命令行访问
2. **xrdp（远程桌面）**：通过 USB、LAN 或 VPN 使用标准 RDP 协议进行图形桌面访问
3. **RustDesk**：使用第三方开源解决方案进行完整的图形桌面访问

每种方法都是独立的——选择适合你需求的方法，或将它们组合使用。

---

## 所需硬件和软件

### 硬件要求

- [Arduino® UNO Q](https://store.arduino.cc/products/uno-q)
- 一台计算机（macOS、Windows 或 Linux）用于连接
- USB-C® 线缆（仅 ADB-over-USB 工作流需要）
- 网络连接（以太网或 Wi-Fi®）用于基于 LAN 或 VPN 的方法

### 软件要求

- [Tailscale](https://tailscale.com/) 帐户（用于 Tailscale + SSH 和 RDP-over-VPN 方法）
- 计算机上的 RDP 客户端（用于 xrdp 方法）—— Windows 预装远程桌面；macOS 使用 [Windows App](https://apps.apple.com/app/windows-app/id1295203466)；Linux 安装 [Remmina](https://remmina.org/)
- 计算机上的 [RustDesk](https://rustdesk.com/) 客户端（用于 RustDesk 方法）

---

## Tailscale + SSH

[Tailscale](https://tailscale.com/) 是一个零配置 VPN，可在你的设备之间创建安全网络。通过在 UNO Q 和你的计算机上安装 Tailscale，你可以从任何地方 SSH 到你的开发板，就像它在你的本地网络上一样。

### 1. 在 UNO Q 上安装 Tailscale

1. 通过 SSH 连接到你的 UNO Q 或在桌面模式下打开终端。

2. 运行以下命令安装 Tailscale：
```bash
curl -fsSL https://tailscale.com/install.sh | sh
```

3. 认证并将开发板连接到你的 Tailscale 网络：
```bash
sudo tailscale up
```

*按照提供的链接使用你的 Tailscale 帐户进行身份验证。*

### 2. 通过 SSH 连接

当你的计算机和 UNO Q 都连接到同一个 Tailscale 网络后，你可以在 Tailscale 管理控制台中找到开发板的 Tailscale IP 地址，或在开发板上运行以下命令：

```bash
tailscale ip -4
```

从你的计算机，使用 Tailscale IP SSH 到开发板：

```bash
ssh arduino@<tailscale-ip>
```

## Xrdp（远程桌面）

[xrdp](http://www.xrdp.org/) 是 Linux 的开源 RDP（远程桌面协议）服务器。RDP 是远程桌面访问的事实标准，RDP 客户端在所有主要平台上都可用——Windows 甚至预装了远程桌面。由于 UNO Q 运行基于 Debian 的操作系统，xrdp 开箱即用，只需最少的配置。

### 1. 在 UNO Q 上安装 Xrdp

通过 SSH 或 ADB shell 连接到你的 UNO Q，然后安装 xrdp 及其依赖项：

```bash
sudo apt update
sudo apt install xrdp dbus-x11
sudo systemctl enable xrdp
sudo systemctl start xrdp
```

验证 `xrdp` 是否正在运行：

```bash
sudo systemctl status xrdp
```

该服务默认监听端口 `3389`。

### 2. RDP 客户端

在你的计算机上安装 RDP 客户端：

- **Windows**：预装远程桌面连接。在开始菜单中搜索 `mstsc` 或"远程桌面连接"来打开它。
- **macOS**：从 App Store 安装 [Windows App](https://apps.apple.com/app/windows-app/id1295203466)（前身为 Microsoft Remote Desktop）。
- **Linux**：安装 [Remmina](https://remmina.org/)：
```bash
sudo apt install remmina
```

### 通过 LAN 进行 RDP（以太网 / Wi-Fi®）

如果 UNO Q 连接到与你的计算机相同的本地网络（通过以太网或 Wi-Fi®），你可以使用开发板的本地 IP 地址直接连接。

1. 在 UNO Q 上，查找其 IP 地址：
```bash
hostname -I
```

> **注意**：`hostname -I` 通常返回多个地址。使用看起来像你 LAN 的那个（通常以 `192.168.` 或 `10.` 开头）。忽略 `172.x.x.x` 地址——这些属于 Docker 的内部网络，不是你的 LAN。

2. 在你的计算机上，打开 RDP 客户端并连接到 `<board-ip>:3389`。

3. 使用你的 UNO Q 凭据登录（默认用户：`arduino`）。

### 通过 USB 进行 RDP（ADB 端口转发）

如果你无法在 UNO Q 上使用网络访问，可以通过 USB 使用 ADB 端口转发来建立 RDP 连接。

1. 通过 USB-C® 将 UNO Q 连接到你的计算机。

2. 确保你的计算机上安装了 ADB（有关安装说明，请参阅 [ADB 教程](https://docs.arduino.cc/tutorials/uno-q/adb/)）。

3. 通过 ADB 转发 RDP 端口：
```bash
adb forward tcp:3389 tcp:3389
```

4. 打开你的 RDP 客户端并连接到 `localhost:3389`。

5. 使用你的 UNO Q 凭据登录（默认用户：`arduino`）。

> **注意**：完成后要停止端口转发，运行 `adb forward --remove tcp:3389`。

### 通过 VPN 进行 RDP（Tailscale）

要从不同网络远程访问 UNO Q 桌面，将 xrdp 与 Tailscale 等 VPN 结合使用。如果你已经按照 [Tailscale + SSH](#tailscale--ssh) 部分设置好了 Tailscale，你可以使用 RDP 客户端连接到开发板的 Tailscale IP。

1. 确保 Tailscale 在 UNO Q 和你的计算机上都在运行。

2. 查找开发板的 Tailscale IP：
```bash
tailscale ip -4
```

3. 打开你的 RDP 客户端并连接到 `<tailscale-ip>:3389`。

4. 使用你的 UNO Q 凭据登录（默认用户：`arduino`）。

这提供了从世界任何地方进行完整的图形桌面访问，而无需将端口 `3389` 暴露给公共互联网。

### 故障排除：连接时黑屏

如果 xrdp 连接后立即显示黑屏并断开连接，原因通常是开发板上已运行的桌面会话冲突。

当 UNO Q 启动时，它会启动一个由 LightDM（登录屏幕和会话管理器）管理的本地桌面会话。该会话已经以 `arduino` 用户登录并使用开发板的主显示器。然后当你通过 xrdp 连接时，它尝试为你的远程连接启动*第二个*桌面会话，但两个会话最终共享同一时间只能由一个会话拥有的资源，最显著的是桌面应用程序用于相互通信的每用户消息总线（D-Bus）。由于本地会话先启动，远程会话无法干净地启动并将你踢出，这就是你看到的黑屏。

解决方法是告诉 xrdp 在完全独立于本地会话的环境中启动其远程桌面会话，使用自己的私有消息总线。覆盖 xrdp 启动脚本：

```bash
sudo tee /etc/xrdp/startwm.sh << 'EOF'
#!/bin/sh
unset DBUS_SESSION_BUS_ADDRESS
unset XDG_RUNTIME_DIR
if [ -r /etc/profile ]; then
    . /etc/profile
fi
exec dbus-run-session -- xfce4-session
EOF
```

然后重启 xrdp：

```bash
sudo systemctl restart xrdp
```

> **注意**：如果你使用不同的桌面环境，请将 `xfce4-session` 替换为适当的命令。使用 `ls /usr/share/xsessions/` 查看可用的会话。

## RustDesk（桌面访问）

[RustDesk](https://rustdesk.com/) 是开源远程桌面软件。由于 UNO Q 可以无头运行（没有物理显示器），我们将配置一个虚拟显示驱动程序，以便桌面环境加载并可以远程访问。

### 1. 安装 RustDesk

首先，在 [RustDesk 发布页面](https://github.com/rustdesk/rustdesk/releases/latest) 上找到最新 `aarch64.deb` 包的链接。

连接到你的开发板（通过 SSH 或终端）并使用 `wget` 下载。然后使用通配符（`*`）安装，使命令无论下载的版本如何都能工作：

```bash
wget <PASTE_DOWNLOAD_LINK_HERE>
sudo dpkg -i rustdesk-*-aarch64.deb
sudo apt -f install
```

### 2. 安装和配置虚拟显示器

1. 安装虚拟显示驱动程序：

```bash
sudo apt install xserver-xorg-video-dummy
sudo mkdir -p /etc/X11/xorg.conf.d
```

2. 创建配置文件：

```bash
sudo tee /etc/X11/xorg.conf.d/10-dummy.conf << 'EOF'
Section "Device"
 Identifier "DummyDevice"
 Driver "dummy"
 VideoRam 256000
 Option "IgnoreEDID" "true"
EndSection

Section "Monitor"
 Identifier "DummyMonitor"
 HorizSync 28.0-80.0
 VertRefresh 48.0-75.0
 Modeline "1920x1080" 148.50 1920 2008 2052 2200 1080 1084 1089 1125
EndSection

Section "Screen"
 Identifier "DummyScreen"
 Device "DummyDevice"
 Monitor "DummyMonitor"
 DefaultDepth 24
 SubSection "Display"
 Depth 24
 Modes "1920x1080"
 EndSubSection
EndSection
EOF
```

### 3. 配置 LightDM 自动登录

LightDM 是 UNO Q 启动时启动桌面会话的登录管理器。默认情况下它显示登录屏幕；我们将配置它自动登录 `arduino` 用户，以便桌面（和 RustDesk）在重启后无需任何用户交互即可使用。

> **警告**：你正在更改开发板的启动方式。应用此步骤后，UNO Q 将直接启动进入 `arduino` 桌面会话，无需输入密码。如果你需要保持登录屏幕启用，请跳过此步骤。

添加自动登录配置（如果你的用户名不同，请替换 `arduino`）：

```bash
sudo tee -a /etc/lightdm/lightdm.conf << 'EOF'
[Seat:*]
autologin-user=arduino
EOF
```

要稍后恢复自动登录，用文本编辑器打开 `/etc/lightdm/lightdm.conf` 并删除刚才添加的两行，然后重启。

### 4. 启用 RustDesk 并重启

启用 RustDesk 服务并重启开发板：

```bash
sudo systemctl enable rustdesk
sudo reboot
```

### 5. 设置 RustDesk 密码

开发板重启后，通过 SSH 重新连接以获取你的 RustDesk ID 并设置密码：

```bash
rustdesk --get-id
sudo rustdesk --password your_password
```

> 记下 ID——你连接时需要它。

### 6. 连接

从 [rustdesk.com](https://rustdesk.com/) 在你的客户端设备（macOS、iOS、Windows、Linux）上安装 RustDesk。输入开发板的 ID 和密码。这可以跨不同网络从世界任何地方工作。

### 在 HDMI 和无头模式之间切换

虚拟驱动程序会覆盖真实的 GPU，这意味着如果你通过 HDMI 插入物理显示器，当虚拟驱动程序处于活动状态时它会显示黑屏。

你可以创建一个脚本轻松在虚拟显示器和物理 HDMI 输出之间切换：

```bash
sudo tee /usr/local/bin/toggle-display << 'EOF'
#!/bin/bash
CONF="/etc/X11/xorg.conf.d/10-dummy.conf"
BAK="${CONF}.bak"

if [ -f "$CONF" ]; then
    mv "$CONF" "$BAK"
    echo "Switched to HDMI (physical display)"
else
    mv "$BAK" "$CONF"
    echo "Switched to dummy (headless/RustDesk)"
fi
systemctl restart lightdm
EOF

sudo chmod +x /usr/local/bin/toggle-display
```

随时通过运行以下命令切换：

```bash
sudo toggle-display
```

### 重启后

一切都会自动启动：LightDM 在虚拟显示器上自动登录，RustDesk 运行，桌面可以从任何设备远程访问。

## 方法比较

| 方法 | 访问类型 | 网络要求 | 优点 | 缺点 |
|------|----------|----------|------|------|
| Tailscale + SSH | 命令行 | 需要 Tailscale 帐户 | 安全、轻量、快速 | 仅命令行访问 |
| xrdp (RDP) | 图形桌面 | LAN/USB/VPN | 标准协议、跨平台客户端 | 需要配置避免会话冲突 |
| RustDesk | 图形桌面 | 互联网连接 | 开源、无需配置网络 | 需要虚拟显示驱动 |

---

## 总结

本教程介绍了三种从本地网络外部远程访问 UNO Q 的方法：

- **Tailscale + SSH**：最适合需要安全命令行访问的场景，设置简单
- **xrdp (RDP)**：最适合需要标准图形桌面访问的场景，支持多种连接方式（LAN/USB/VPN）
- **RustDesk**：最适合需要完全开源解决方案和无头操作的场景

根据你的具体需求选择合适的方法，或组合使用以满足不同场景的要求。
