## 1. 系统要求

### 硬件
- Arduino UNO Q 开发板
- USB-C 数据线
- Ubuntu 计算机

### 软件
- Arduino App Lab（可选，用于图形化开发）
- ADB（Android Debug Bridge，用于命令行连接）

### 网络要求
- 首次设置需要网络连接
- 后续运行 Apps 不强制需要网络（除非使用云端功能）

## 2. USB 权限配置

**Ubuntu 用户必须先配置 USB 权限，否则无法正确识别开发板。**

### 方法一：自动安装（推荐）

```bash
cd ~/Downloads
wget https://raw.githubusercontent.com/arduino/ArduinoCore-zephyr/main/post_install.sh
chmod +x post_install.sh
sudo ./post_install.sh
```

### 方法二：手动配置

创建 udev 规则文件：

```bash
echo \
'# Operating mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="2341", ATTRS{idProduct}=="0078", MODE="0660", TAG+="uaccess"
# EDL mode
SUBSYSTEMS=="usb", ATTRS{idVendor}=="05c6", ATTRS{idProduct}=="9008", MODE="0660", TAG+="uaccess"' \
| sudo tee "/etc/udev/rules.d/60-Arduino-UNO-Q.rules" \
&& sudo udevadm control --reload-rules \
&& sudo udevadm trigger
```

### 验证权限配置

```bash
# 检查规则文件是否创建成功
cat /etc/udev/rules.d/60-Arduino-UNO-Q.rules

# 连接开发板后检查是否被识别
lsusb | grep -E "2341:0078|05c6:9008"

# 检查 ADB 访问权限
adb devices
```

**重要提示：** 配置完成后，**拔掉开发板，等待几秒后重新连接**，让新权限生效。

## 3. 连接方式选择

UNO Q 支持三种连接模式：

| 模式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **USB 桌面模式** | 开发调试 | 最简单，功能最全 | 需要保持 USB 连接 |
| **网络模式 (SSH)** | 远程部署 | 无需物理连接 | 需在同一局域网 |
| **SBC 模式** | 独立运行 | 完全独立 | 需要额外硬件 |

**推荐新手使用 USB 桌面模式。**

---

## 4. 方式一：USB 桌面模式（推荐）

### 步骤 1：安装 Arduino App Lab

```bash
# 下载 Arduino App Lab
# 访问 https://www.arduino.cc/en/software/#app-lab-section

# 解压下载的文件
cd ~/Downloads
tar -xf ArduinoAppLab*.tar.gz
mv ArduinoAppLab*/ ~/Desktop/ArduinoAppLab

# 安装依赖
sudo apt install libwebkit2gtk-4.1-0
```

### 步骤 2：启动 Arduino App Lab

```bash
cd ~/Desktop/ArduinoAppLab
./arduino-app-lab
```

### 步骤 3：连接开发板

1. 使用 USB-C 数据线将 UNO Q 连接到计算机
2. 等待开发板启动完成（LED 矩阵动画停止表示启动完成）
3. 在 Arduino App Lab 中选择 **USB** 模式

### 步骤 4：首次设置

首次连接时，按提示完成：
1. 配置 Wi-Fi 网络
2. 设置开发板名称和密码
3. （可选）登录 Arduino 账户

---

## 5. 方式二：网络模式（SSH）

**前提条件：** 开发板必须先通过 USB 完成首次设置（包括 Wi-Fi 配置）。

### 查找开发板 IP 地址

**方法一：通过 ADB 查询**

```bash
adb shell ip addr show wlan0
```

**方法二：通过路由器管理界面**

查看已连接设备的 IP 列表，找到名为 `arduino-uno-q` 的设备。

**方法三：使用 nmap 扫描**

```bash
# 扫描本地网段
nmap -sn 192.168.1.0/24 | grep -B 2 "Arduino"
```

### SSH 连接

```bash
# 默认用户名：arduino
ssh arduino@<开发板IP地址>

# 首次连接会提示确认指纹，输入 yes
# 密码：首次 USB 设置时创建的密码
```

### 常用 SSH 操作

```bash
# 查看系统信息
cat /etc/os-release

# 查看 CPU 温度
cat /sys/class/thermal/thermal_zone0/temp

# 查看内存使用
free -h

# 查看磁盘使用
df -h

# 安全关机
sudo shutdown -h now

# 重启
sudo reboot
```

## 6. 方式三：SBC 模式

### 所需硬件
- USB-C 转接器（带 HDMI 和 USB 接口）
- HDMI 显示器和数据线
- USB 键盘和鼠标
- 5V/3A 电源适配器

### 连接步骤

1. 将 USB-C 转接器连接到 UNO Q 的 USB-C 端口
2. 连接 HDMI 显示器
3. 连接键盘和鼠标
4. 使用 5V/3A 电源为转接器供电
5. 等待开发板启动，出现登录界面
6. 登录：
   - 用户名：`arduino`
   - 密码：首次设置时创建的密码

登录后，Arduino App Lab 会自动启动。

## 7. 运行第一个 App

### 使用 Arduino App Lab

1. 打开 Arduino App Lab
2. 在左侧菜单选择 **Examples**
3. 选择一个示例，如 **Blink**（LED 闪烁）
4. 点击右上角 **Run** 按钮
5. 等待编译和部署完成
6. 观察 LED 矩阵上的效果

### 查看运行日志

在 Console 面板中：
- **Start-up**：启动日志
- **Main (Python®)**：Python 应用输出
- **Sketch (Microcontroller)**：微控制器输出

**注意：** 使用 `Monitor.print()` 而非 `Serial.println()` 才能在 Console 中看到输出。

## 8. 常见问题排查

### 问题 1：ADB 提示 "insufficient permissions"

**原因：** USB 权限未正确配置

**解决方案：**
```bash
# 重新安装 udev 规则
sudo ./post_install.sh

# 重启 udev 服务
sudo systemctl restart systemd-udevd

# 拔掉并重新连接开发板
```

### 问题 2：开发板未被识别

**排查步骤：**
```bash
# 检查 USB 连接
lsusb | grep -E "2341|05c6"

# 检查 ADB 设备
adb devices

# 如果为空，尝试重启 ADB 服务
adb kill-server
adb start-server
```

### 问题 3：无法连接 Wi-Fi

**解决方案：**
1. 确保 USB 连接稳定
2. 重新通过 Arduino App Lab 配置 Wi-Fi
3. 检查 Wi-Fi 密码是否正确
4. 确认 Wi-Fi 网络是 2.4GHz（部分开发板不支持 5GHz）

### 问题 4：SSH 连接被拒绝

**排查步骤：**
```bash
# 确认开发板和计算机在同一网络
ping <开发板IP>

# 检查 SSH 服务状态
ssh arduino@<IP> "systemctl status ssh"

# 如需重启 SSH 服务
ssh arduino@<IP> "sudo systemctl restart ssh"
```

### 问题 5：App 运行失败

**排查步骤：**
1. 查看 Console 中的错误日志
2. 检查 Python 语法错误（Main 选项卡）
3. 检查 Sketch 编译错误（Start-up 选项卡）
4. 尝试运行官方示例确认环境正常

### 问题 6：Linux 下 qcserial 模块冲突

**症状：** 刷写镜像时工具意外退出

**解决方案：**
```bash
# 检查 qcserial 是否加载
lsmod | grep qcse

# 如果存在，将其加入黑名单
sudo nano /etc/modprobe.d/blacklist-modem.conf

# 添加以下内容：
# blacklist qcserial

# 保存后重启系统
sudo reboot
```

## 快速命令参考

```bash
# ADB 常用命令
adb devices              # 列出已连接设备
adb shell                # 进入 Shell
adb shell ip addr        # 查看 IP 地址
adb push <file> <dest>   # 推送文件到开发板
adb pull <file> <dest>   # 从开发板拉取文件

# 系统信息
cat /etc/os-release      # 系统版本
uname -a                 # 内核信息
free -h                  # 内存使用
df -h                    # 磁盘使用
cat /sys/class/thermal/thermal_zone0/temp  # CPU 温度

# 网络管理
nmcli device status      # 网络设备状态
nmcli connection show    # 已配置连接
nmcli device wifi list   # 扫描 Wi-Fi

# 软件包管理
sudo apt update          # 更新软件源
sudo apt upgrade         # 升级软件包
sudo apt install <pkg>   # 安装软件包

# 电源管理
sudo shutdown -h now     # 关机
sudo reboot              # 重启
```
