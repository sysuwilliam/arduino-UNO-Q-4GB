# 理解 Bricks：Arduino App Lab 中的构建块

学习 Bricks 以及它们如何用于构建 Apps

**作者**：Karl Söderby  
**最后修订**：2025年7月10日

---

Bricks 是用于在 [Arduino App Lab](https://docs.arduino.cc/software/app-lab) 中构建 [Apps](https://docs.arduino.cc/software/app-lab/tutorials/getting-started/) 的**代码构建块**。Bricks 具有广泛的应用范围，有些嵌入 AI 模型，而其他 Bricks 用于托管 Web 应用程序或连接外部 Web API。

它们旨在通过抽象代码并暴露可在 App 中使用的易于使用的 API，使构建复杂 Apps（如嵌入**计算机视觉**）变得更容易。一个 App 可以有多个 Bricks，可能包括：

- AI 模型，例如用于目标检测或图像识别
- Web UI 构建器，用于托管 Web 界面以交互或显示来自开发板的数据
- REST API 处理器，可以通过单个命令返回天气数据

在本指南中，我们将探讨 Bricks 如何工作、它们如何在 Apps 中使用，以及可用的不同 Bricks。

***每个 Brick 的 API 文档可以在 Arduino App Lab 内的"Bricks"选项卡中访问。***

## 理解 Brick 概念

![Motion detection example](https://docs.arduino.cc/static/75a33602f9eaea2f7f7ce68e8a65301a/a6d36/motion-detection.png)

Bricks 是导入到 App 中的代码包，并在 Linux 系统上作为独立进程启动。Bricks 使用 **Python** 编写。在 UNO Q 上启动 App 时，Bricks 被部署到开发板上，并且可能使用 [docker](https://www.docker.com/) 容器，可以通过易于使用的 API 从主应用程序进行交互。

首先在 UI 中添加 Bricks，方法是点击按钮（见下图），然后选择我们要在 App 中使用的 Brick。

![Importing a Brick](https://docs.arduino.cc/static/c4ba129c26c39fd5f37d2dfed7fee780/a6d36/add-brick-1.png)

![Importing a Brick](https://docs.arduino.cc/static/9634172e298c5e8202cb098004b4e6da/a6d36/add-brick-2.png)

添加 Brick 也会更新 `app.yaml` 文件：

```
bricks:
    - arduino:web_ui
    - arduino:motion_detection
```

***不要手动编辑 `app.yaml` 文件中的 `bricks` 条目。***

将 Brick 添加到 App 后，还需要在 App 的 `python` 文件夹内的 `main.py` 文件中导入它。以下是示例：

```
from arduino.app_bricks.web_ui import WebUI
from arduino.app_bricks.motion_detection import MotionDetection
```

- 在上面的示例中，`WebUI` 和 `MotionDetection` 类从 `web_ui` 和 `motion_detection` Bricks 导入。

要访问 Brick 的功能，我们使用：

```
motion_detection = MotionDetection(confidence=CONFIDENCE)
web_ui = WebUI()
```

***要更好地理解每个 Brick 的作用，请参阅 Arduino App Lab 中的"Bricks"部分。***

### Brick 类示例

其他 Bricks 更高级，例如 `objectdetection` Brick。此 Brick 可以检测图像中的对象，并使用 **YoloX Nano** 模型返回类别标签和置信度分数。

```
object_detection = ObjectDetection()
frame = os.read("path/to/your/image.jpg")

out = object_detection.detect(frame)
obj_det = out["detection"][0]
detected_object = obj_det.get("class_name", None)
```

在 Brick 内部，`detect` 方法如下所示：

```
def detect(self, image_bytes, image_type: str = "jpg", confidence :float = None) -> dict:
        """Process an image to detect objects.

        Args:
            image_bytes: can be raw bytes or PIL image.
            image_type: type of image (jpg, jpeg, png). Default is jpg.
            confidence: confidence level for detection. Default is None (use module defaults).
        """
        if not image_bytes or not image_type:
            return None
        ret = super().infer_from_image(image_bytes, image_type)
        return self._extract_detection(ret, confidence)
```

## 有哪些 Bricks 可用？

有许多 Bricks 可供选择。Bricks 分为以下类别：

- **AI - Audio**（AI - 音频）
- **AI - Computer Vision**（AI - 计算机视觉）
- **AI - Sensor data**（AI - 传感器数据）
- **API**
- **IoT**
- **Storage**（存储）
- **Web User Interface**（Web 用户界面）

要查看完整的 Bricks 列表，请导航到 Arduino App Lab 中的 **"Bricks"** 选项卡。描述、用法、API 文档和链接示例也可以在此部分找到。

## Brick 类型

Bricks 的实现级别各不相同，有些 Bricks 的工作方式类似于 Python® 模块，其他则使用 **Docker** 在开发板上启动容器。部署 Bricks 由 Arduino App Lab 自动处理。

一些 Bricks（如 `web_ui`）使用 App 内的 `assets` 文件夹，其中可以配置 HTML/CSS/JS 文件以通过 WebSocket 与 Python 应用程序交互。这个特定的 Brick 还在 `http://localhost:7000` 启动其 Web 应用程序，同一网络上的其他设备可以使用 `<board-name>.local:7000` 地址访问。

开发板名称显示在 Arduino App Lab 底部，是我们首次设置时设置的名称。

## 带有 AI 模型的 Bricks

嵌入 AI 模型的 Bricks 使用 Docker 容器启动 Brick，该容器从主 Python® 应用程序调用。这些 Bricks 使用预先配置的默认模型。

使用 AI 模型的 Bricks 使用 [Docker](https://www.docker.com/) 容器部署。这会在开发板上生成一个独立进程，可以通过 Python® 应用程序的 API 进行交互。

这可以在 App 的**启动**期间通过检查日志来检测。

```
# Container for Brick started
Container local-share-arduino-app-cli-examples-real-time-accelerometer-ei-motion-detection-runner-1 Started
# Container for App started
Container local-share-arduino-app-cli-examples-real-time-accelerometer-main-1 Started
```

### Docker 状态

可以通过在开发板的终端中运行 `docker ps` 来查看 Docker 容器的状态。要访问终端，我们可以：

- 在[单板计算机（SBC）](https://docs.arduino.cc/tutorials/uno-q/single-board-computer)模式下打开终端并运行 `docker ps`
- 通过 `adb` 访问开发板的 Shell（开发板通过 USB 连接）

运行命令提供有关容器的信息，如果在启动 App 期间或之后发生问题，可以查看它是否正在运行。

运行命令的输出如下所示：

