# Arduino App CLI：从命令行管理 Apps

学习如何使用 Arduino App Lab CLI，这是用于在 UNO Q 开发板上启动 Apps 的命令行工具。

**作者**：Karl Söderby  
**最后修订**：2025年11月20日

---

Arduino App Lab 构建在 `arduino-app-cli` 工具之上，这是一个用于管理为 [Arduino UNO Q](https://store.arduino.cc/products/uno-q) 开发板制作的 Apps 的命令行工具。

`arduino-app-cli` 已预装在 UNO Q 开发板上，可以通过开发板本身的终端访问，也可以通过 Android Debug Bridge（ADB）从主机计算机访问开发板的终端。

---

## 要求

需要以下硬件：

- [Arduino UNO Q](https://store.arduino.cc/products/uno-q)
- [USB-C® 类型数据线](https://store.arduino.cc/products/usb-cable2in1-type-c)

要通过 `adb`（通过 USB）访问开发板，您还需要安装以下软件：

- [Android Debug Bridge](https://developer.android.com/tools/releases/platform-tools)

您也可以通过 SSH 访问开发板，SSH 通常默认安装在您的系统上。

---

## 通过 ADB 连接

***要了解更多关于设置 `adb` 的信息，请查看 [通过 ADB 连接 UNO Q](https://docs.arduino.cc/tutorials/uno-q/adb/) 教程。本指南将引导您完成安装步骤。***

1. 通过 USB-C 将 UNO Q 开发板连接到计算机。

2. 在终端中运行 `adb devices`。这应该列出已连接的设备。

![Connected devices](https://docs.arduino.cc/static/3539f1152db188f36d4b0c1460905bd4/a6d36/connected-devices.png)

请注意，连接设备后可能需要长达一分钟才能显示。

3. 运行 `adb shell`。

4. 您现在应该在开发板的终端内。

![Terminal on the board](https://docs.arduino.cc/static/1ff569c222a5d6530fee3b21dd66e495/a6d36/board-terminal.png)

5. 您现在可以通过开发板上的终端运行命令！要退出终端，只需输入 `exit`。

**注意：** 如果您之前没有通过 Arduino App Lab 设置开发板，第一次运行需要身份验证的命令（如 `sudo`）时，系统会提示您创建新密码。

---

## 通过 SSH 连接

***注意：要使用 SSH，需要完成[首次设置](https://docs.arduino.cc/software/app-lab/tutorials/getting-started#install--set-up-arduino-app-lab)。这是通过 USB 连接开发板、打开 Arduino App Lab 并选择 USB 选项来完成的。在这里，您需要为开发板命名、设置新密码并提供 Wi-Fi® 凭据。SSH 将在此设置期间自动配置。***

1. 在计算机上打开终端。

2. 运行 `ssh arduino@<boardname>.local`

3. 提供开发板的密码。

4. 我们现在可以远程与开发板的 Shell 交互！✅

![Connect via SSH](https://docs.arduino.cc/static/66329408af0db4d2890646f3734a961e/a6d36/ssh-macos.png)

- 有关通过 SSH 连接到 UNO Q 的更详细教程可在[这里](https://docs.arduino.cc/tutorials/uno-q/ssh)找到。

## 使用 Arduino App CLI

使用 `arduino-app-cli` 工具，您可以：

- 启动/停止 Apps
- 列出正在运行的 Apps
- 创建新 Apps
- 显示 App 的日志
- 监控 App

要全面了解可用命令，请在终端中输入 `arduino-app-cli`。

### 创建 App

要管理 Apps，我们使用 `app` 命令。

要创建 App，我们可以使用：

```
arduino-app-cli app new "test"
```

这将在 `/home/arduino/ArduinoApps/test` 创建一个 App，包含配置文件以及 sketch 和 Python® 文件夹。

### 编辑 App

如果您使用带有显示器、键盘和鼠标的开发板，可以在代码编辑器中打开文件，例如 *Vim*、*gedit* 或 *Sublime*。

如果您通过 `adb` 访问开发板，可以从主机计算机**拉取**和**推送**文件/文件夹。

要拉取文件，使用：

```
adb pull /home/arduino/ArduinoApps /path/to/localfolder
```

要推送文件，使用：

```
adb push /path/to/localfolder /home/arduino/ArduinoApps
```

**注意：** 您可能需要授予 `ArduinoApps` 文件夹的权限。可以通过运行 `adb shell chown -R arduino:arduino /home/arduino/ArduinoApps` 来完成。

### 启动和停止 Apps

一旦创建并编辑了 App，可以通过以下命令启动：

```
arduino-app-cli app start "/home/arduino/ArduinoApps/test"
```

这将在您的 UNO Q 开发板上启动 App。

要停止 App，使用：

```
arduino-app-cli app stop "/home/arduino/ArduinoApps/test"
```

### 读取 App 日志

要监控运行中 App 的日志，使用 `logs` 命令：

```
arduino-app-cli app logs /home/arduino/ArduinoApps/test --all
```

这将列出 App 的日志：

![View the logs of an App](https://docs.arduino.cc/static/8197c7684de087d01b2086d3217ae011/a6d36/app-logs.png)

## 运行示例和用户 Apps

要运行内置示例和我们创建的 Apps，我们可以使用 `user` 和 `examples` 快捷方式（而不是指定路径）。

```
# 运行您自己的 app
arduino-app-cli app start user:my-app 

# 运行示例 app（例如 blink）
arduino-app-cli app start examples:blink
```

### 列出 Apps

要列出可用的 Apps，使用 `app list` 命令。

```
arduino-app-cli app list
```

这将列出所有可用的 Apps（包括示例）及其状态：

![List Apps](https://docs.arduino.cc/static/b8752ee8ae942ee03e3d452945ef4b95/a6d36/list-apps.png)

## 系统配置和更新

`system` 命令允许您管理开发板上的系统配置和更新。

要检查更新，运行：

```
arduino-app-cli system update
```

这将提示您安装任何可用的更新。

要设置开发板名称，使用：

```
arduino-app-cli system set-name "my-board"
```

这将更改开发板的名称，在重置开发板后生效。

要启用或禁用网络模式，使用：

```
arduino-app-cli system network enable/disable
```

网络模式将启用 SSH，并允许客户端通过本地网络连接到开发板。

最后，您可以通过运行以下命令清理未使用的容器和镜像来回收存储空间：

```
arduino-app-cli system cleanup
```

## Bricks

目前，只能列出可用的 Bricks 和每个 Brick 的具体详细信息。

这通过运行以下命令完成：

```
# 列出开发板上安装的 Bricks
arduino-app-cli brick list

# 特定 Brick 的详细信息
arduino-app-cli brick details arduino:<brick>
```

这将显示类似以下内容：

![List Bricks](https://docs.arduino.cc/static/8aaa0a53a418a199cdf5ceedb20c31a7/a6d36/list-bricks.png)
