# `weather_frames.h` 解析

## 原文件作用

`weather_frames.h` 保存的是 LED 点阵动画资源。

它的本质不是逻辑代码，而是“显示素材数据”。Arduino 程序通过加载这些数组，把不同天气转换成对应的动画效果。

## 文件结构

文件中定义了 5 组天气动画数组：

- `sunny`
- `cloudy`
- `rainy`
- `snowy`
- `foggy`

每个数组都是二维数组，类型为：

```cpp
const uint32_t name[][5]
```

这说明每一帧包含 5 个 `uint32_t` 数值。

## 每一帧的数据含义

从 Arduino LED Matrix 常见用法推测，每一帧中：

- 前 4 个数值用于编码点阵图案；
- 第 5 个数值表示该帧显示的时长。

本项目中这一点非常明显，因为最后一个值都像时间参数，例如：

- `500`
- `200`
- `650`
- `660`

这些数值正好与不同天气动画的节奏相匹配。

## 各天气动画特征

### `sunny`

- 3 帧
- 每帧 500ms
- 第一帧和第三帧相同

这类结构通常用于制造轻微闪烁或往返式动态效果。

### `cloudy`

- 4 帧
- 每帧 500ms

帧数比晴天更多，说明云朵的轮廓变化更细一些，可以表现漂移感。

### `rainy`

- 3 帧
- 每帧 200ms

速度明显更快，符合“雨滴下落较急”的视觉印象。

### `snowy`

- 3 帧
- 每帧 650ms

节奏更慢，更符合雪花飘落的观感。

### `foggy`

- 3 帧
- 每帧 660ms

变化很慢，适合表现雾气缓慢流动。

## 与 `sketch.ino` 的关系

Arduino 主程序中会根据天气分类加载对应序列：

```cpp
matrix.loadSequence(sunny);
matrix.loadSequence(cloudy);
matrix.loadSequence(rainy);
...
```

也就是说：

- `weather_frames.h` 提供动画资源；
- `sketch.ino` 决定什么时候加载哪一组资源；
- `matrix.playSequence()` 负责把资源播放出来。

## 为什么这个文件要单独存在

把动画帧单独放在头文件里有几个优点：

- 让主逻辑文件更简洁；
- 便于单独维护图案资源；
- 如果后续要新增天气图案，不需要改动主流程代码；
- 逻辑和资源分离，工程结构更清晰。

## 小结

`weather_frames.h` 是本项目实现可视化效果的核心资源文件。

没有这个文件，项目仍然可以获得天气分类，但无法在 LED 点阵上播放具体的天气动画。
