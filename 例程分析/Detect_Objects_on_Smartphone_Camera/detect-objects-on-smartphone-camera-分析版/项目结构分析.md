# 项目结构分析

## 目录结构

原始项目的核心结构如下：

```text
detect-objects-on-smartphone-camera/
├── README.md
├── app.yaml
├── python/
│   └── main.py
└── assets/
    ├── index.html
    ├── app.js
    ├── style.css
    ├── libs/
    ├── img/
    └── docs_assets/
```

## 各部分职责

- `README.md`
  官方说明文档，介绍用途、运行方式和整体工作原理。

- `app.yaml`
  App Lab 应用元数据配置，定义应用名称、暴露端口和所用 Bricks。

- `python/main.py`
  后端主程序，负责远程摄像头初始化、目标检测和 Web UI 消息通信。

- `assets/index.html`
  前端页面结构，定义视频区域、控制区、反馈区和最近检测列表。

- `assets/app.js`
  前端主逻辑，负责二维码生成、Socket.IO 通信、视频状态切换、结果渲染和阈值控制。

- `assets/style.css`
  页面样式文件。

- `assets/libs/`
  前端依赖库，如 `socket.io.min.js`、`qrcode.min.js`。

- `assets/img/`
  前端图标和反馈图像资源。

## 架构划分

这个示例的架构可以分成四层：

### 1. 手机输入层

作用：

- 提供实时视频流
- 通过 Arduino IoT Remote 与开发板建立连接

特点：

- 手机不是本地 USB 摄像头，而是远程网络摄像头
- 配对过程通过二维码完成

### 2. Python 后端层

作用：

- 管理摄像头对象
- 调用 AI Brick 执行检测
- 与前端页面实时通信

特点：

- 是整个应用的调度核心
- 不直接处理 HTML，但负责提供数据和状态

### 3. Bricks 能力层

作用：

- `web_ui` 负责网页交互
- `video_objectdetection` 负责 AI 目标检测

特点：

- 这一层提供高级能力
- 让项目不需要自己手写模型推理和网页服务底座

### 4. 前端页面层

作用：

- 展示二维码
- 展示视频流
- 展示检测结果
- 提供阈值调节界面

特点：

- 面向用户交互
- 不承担模型推理
- 所有检测结果都来自 Python 后端推送

## 运行流程

这个项目启动时，整体流程如下：

1. Python 启动。
2. 生成一个 6 位数字密钥。
3. 初始化 `WebUI`。
4. 初始化 `WebSocketCamera`。
5. 初始化 `VideoObjectDetection`。
6. 浏览器访问 Web UI。
7. Python 向前端发送配对信息。
8. 前端生成二维码。
9. 手机扫码并与开发板建立连接。
10. 手机视频流传给开发板。
11. `VideoObjectDetection` 对视频进行推理。
12. Python 将检测结果发送给前端。
13. 前端显示检测结果和反馈动画。

## 调用链分析

从功能调用角度，可以概括为：

```text
App.run()
-> WebUI
-> WebSocketCamera
-> VideoObjectDetection
-> ui.send_message(...)
-> Socket.IO 前端
-> 页面更新
```

从配对流程角度，可以概括为：

```text
Python 生成 secret
-> 前端接收 welcome 消息
-> 前端生成二维码
-> 手机扫码
-> 手机连接 UNO Q
-> 前端进入 streaming 状态
```

## 设计特点

### 优点

- 结构清晰，前后端职责分离
- 手机输入让项目更灵活，不依赖 USB 摄像头
- 使用 Brick 能快速搭建 AI 应用
- Web 界面交互完整，适合展示和扩展

### 局限

- 强依赖同一局域网环境
- 依赖手机 App 配对流程
- 对网络稳定性有要求
- 前端和后端逻辑都较多，阅读门槛高于简单例程

## 和传统 Arduino 项目的差别

这个项目与传统 Arduino 项目的最大差别在于：

- 没有单一的 `.ino` 作为核心
- 核心逻辑集中在 Python 和 Web 前端
- 使用的是 UNO Q 的 App Lab 运行模型，而不是经典单片机式开发

这说明 UNO Q 的开发方式更接近“嵌入式全栈应用”，而不是单纯的“写一个 Arduino 草图”。
